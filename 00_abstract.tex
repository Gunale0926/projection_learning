Runtime learning systems often treat prediction error as a signal to update state or parameters. We argue that the first-order question is structural: is the error repairable under the current information, or does it require a new distinction? Let $\calF_t$ be the information available to a runtime predictor $f_t$, let $\calG\supseteq\calF_t$ be a candidate refinement, and let $R_L(\calF)$ denote the Bayes risk achievable by predictors measurable with respect to $\calF$. We decompose runtime excess risk into
\[
\Delta_{\mathrm{repair}}(f_t,\calF_t)=R_L(f_t)-R_L(\calF_t),
\qquad
\Delta_{\mathrm{refine}}(\calF_t,\calG)=R_L(\calF_t)-R_L(\calG).
\]
The first term is the residual error of the current predictor at fixed information; the second is the value of information absent from the current representation.

Using information maps, finite partitions, and sigma-algebras, we prove an architecture-independent obstruction: if two worlds have the same current information but require different consequences, no repair that preserves the information map can satisfy both. The minimal corrective operation is canonical, namely the join of the current information with the missing evidence. We instantiate this principle as Distinction Graph Machines (DGM), a runtime learner in which nodes carry repairable local memories and edges generate the distinctions used by future reads. For language models, DGM-Embedding implements the same refine-or-repair rule as a detached next-token memory layer with differentiable reads and forward-only writes. Controlled hidden-regime sequence experiments isolate the predicted behavior: repair-only memories collapse incompatible consequences, whereas refine-and-repair DGM creates runtime concepts and recovers the needed distinctions.
