Runtime learning errors are not all the same. Some errors mean that the model is using the right concept but its local state is stale; others mean that the concept itself is too coarse and merges situations with different consequences. We formalize this distinction. A learner's information state is a partition or sigma-algebra, and admissible predictions are measurable with respect to it. If two states in the same atom require different outputs, no local repair or optimizer over the current information can solve the contradiction; the information structure must be refined.

We introduce Distinction Graph Machines, a runtime-learning model that implements this refine-or-repair principle. A DGM maintains anchored runtime concepts, movable retrieval centroids, local memories, and pairwise distinction edges. Edges are binary generators: they define observable distinctions and gate future routing. When new evidence arrives, DGM either repairs a local memory or creates a new anchored concept and edges. We prove that minimal refinement is the smallest information update making evidence measurable, that refinement value equals conditional mutual information under log loss and conditional variance reduction under squared loss, and that guarded refine-or-repair decreases a penalized local objective. We instantiate the model as DGM-Embedding for language modeling, where hidden-state concepts store next-token memories and update by a sequential forward-only rule without inference-time backpropagation. Projection Memory and CPM appear as local repair mechanisms once the relevant distinction structure has been fixed.
