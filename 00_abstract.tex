Runtime learners often treat every surprise as an update signal. This paper argues that the first question should instead be which part of runtime excess risk is being observed. Given current information $\calF_t$, predictor $f_t$, a candidate refinement $\calG\supseteq\calF_t$, and Bayes risk $R^\star$ under an information structure, excess risk decomposes into a repair gap $R(f_t)-R^\star(\calF_t)$ and a refinement gap $R^\star(\calF_t)-R^\star(\calG)$. The first is stale local state under adequate information; the second is missing information.

We formalize this decomposition with information maps, partitions, and sigma-algebras, prove the architecture-independent missing-distinction obstruction, and show that the minimal corrective refinement is the join with the missing evidence. We then introduce Distinction Graph Machines (DGM), a runtime learner whose nodes repair local memories and whose edges create the distinctions that future reads may condition on. For language models, DGM-Embedding implements the same refine-or-repair rule as a detached, forward-only next-token memory layer with no inference-time backpropagation. Controlled hidden-regime sequence experiments show the intended separation: repair-only memories collapse incompatible consequences, while refine+repair DGM creates runtime concepts and recovers the needed distinctions.
