\section{Distinction Graph Machines: A Data Structure for Refinable Partitions}
\label{sec:algorithms}

DGM implements refinable information structures as a graph: edges generate distinctions and nodes store repairable memories. The theory so far gives a diagnostic: decide whether an observed error is better explained by $\Delta_{\mathrm{repair}}$ at fixed information or by $\Delta_{\mathrm{refine}}$ from missing distinctions. A runtime implementation therefore needs a state representation that can both repair local statistics and add distinctions. DGM is the data-structure realization of the abstract refinable-partition model.

\subsection{Why a graph, not a tree?}

\begin{definition}[DGM concept]
\label{def:dgm-concept}
A DGM concept is a memory location selected by a graph-generated hard-route map and equipped with a local memory. It is not claimed to be a human-interpretable semantic concept and need not correspond one-to-one with an atom of the edge partition. Local memories live at graph nodes, and distinction predicates live on edges between nodes that have been confused.
\end{definition}

The central object is
\begin{align*}
\text{DGM}=\text{graph-generated information structure}+\text{local memory repair}.
\end{align*}
The graph is not metadata: its edges generate a finite edge-skeleton information structure and gate the read path, while anchor order completes the exact hard route. The partition and generated-sigma-algebra language follows standard finite measurable-space constructions \citep{billingsley1995probability,kallenberg2002foundations}. The main-text statements below give the operational theorem forms; full proofs are in Appendix~\ref{app:algorithm-bridge-proofs}.

\begin{definition}[Hard-DGM and Soft-DGM]
\label{def:hard-soft-dgm}
Hard-DGM is the formal algorithm: fixed anchors and accepted binary edges generate an edge skeleton, while the exact hard route is measured by the join of this edge skeleton with the anchor-distance ordering. A repair changes only local actions or memories and leaves the hard-route information fixed; a refinement adds an anchored concept, new edge generators, and the anchor-order events involving the new anchor. Soft-DGM is the practical relaxation used for sparse neural hidden states: movable centroids retrieve candidates, smooth edge gates approximate hard eligibility, and routing weights mix local memories.
\end{definition}

The partition-level guarantees are exact for Hard-DGM. Soft-DGM is covered only as an approximation under the candidate-recall and margin conditions in Theorem~\ref{thm:soft-routing-hard-limit:informal}.

\subsection{Hard-DGM state and invariants}

\begin{definition}[Hard-DGM state and anchor edge]
\label{def:hard-dgm-state}
Let $\phi_\theta:\calX\to\R^d$ be an encoder, write $h=\phi_\theta(x)$, and let $\mathsf H\subseteq\R^d$ denote the hidden-state space on which the runtime structure is evaluated. A Distinction Graph Machine has runtime state
\begin{align*}
\calS_t=\left(\{a_j,c_j,M_j,n_j\}_{j=1}^{N_t},G_t\right),
\qquad
G_t=(V_t,E_t),\qquad V_t=\{1,\ldots,N_t\}.
\end{align*}
Each component has a role forced by the refine-or-repair split. A runtime concept needs a stable identity, so $a_j\in\R^d$ is a frozen distinction anchor. Retrieval may adapt as examples arrive, so $c_j\in\R^d$ is a movable centroid used only by the soft implementation. State inside an adequate concept must be repairable, so $M_j$ is a local memory and $n_j$ is a usage count. If a concept is created because it was confused with existing concepts, that conflict becomes an edge in $E_t$. A new concept is initialized with $a_j=c_j=h$. Subsequent repair may move $c_j$, but anchors remain fixed once created, so the hard information structure is unchanged by centroid repair.

An edge $e=(i,j)$ stores a linear distinction
\begin{align*}
u_e=\frac{a_i-a_j}{\norm{a_i-a_j}_2},\qquad b_e=\frac{1}{2}u_e^\top(a_i+a_j),
\end{align*}
with side labels $s_{e,i}=+1$ and $s_{e,j}=-1$. Its binary generator is
\begin{align*}
g_e(h)=\one\{u_e^\top h>b_e\}.
\end{align*}
\end{definition}

\begin{definition}[Graph signature and edge-generated partition]
\label{def:graph-signature-partition}
The edge set induces the \emph{edge skeleton} information structure on $\mathsf H$. The graph signature is
\begin{align*}
\sigma_t^G(h)=\big(g_e(h)\big)_{e\in E_t}\in\{0,1\}^{E_t}.
\end{align*}
Two hidden states are indistinguishable by the current graph if their signatures agree. The nonempty level sets
\begin{align*}
A_a^t=\{h\in\mathsf H:\sigma_t^G(h)=a\},\qquad a\in\{0,1\}^{E_t},
\end{align*}
form a finite partition $\calP_t^G$ of hidden-state space, and the generated sigma-algebra is $\sigma(\calP_t^G)$. Adding a new edge is therefore a join with one binary generator:
\begin{align*}
\calP_{t+1}^G=\calP_t^G\vee\sigma(g_e).
\end{align*}
\end{definition}

This edge partition is only the skeleton. The exact hard route also needs the ordering of eligible anchors by anchor distance; that order is added below as $\calF_t^{\mathrm{anchor\text{-}order}}$. Centroids and local memories are adaptive statistics attached to hard-route states or sparse neighborhoods, not generators of the exact Hard-DGM information structure. Concept nodes and graph-generated partition atoms are related but not identical: a node is a data-structure representative with memory and retrieval statistics; the formal information structure is generated by edge signatures and anchor-order events.

\begin{theorem}[Anchor distinction separates its endpoints]
\label{thm:anchor-edge:informal}
If $a_i\ne a_j$ and $u_e,b_e$ are defined as above for edge $e=(i,j)$, then
\begin{align*}
u_e^\top a_i-b_e=\frac{1}{2}\norm{a_i-a_j}_2,
\qquad
u_e^\top a_j-b_e=-\frac{1}{2}\norm{a_i-a_j}_2.
\end{align*}
Thus each edge distinction separates the two anchors that caused it.
\end{theorem}

Implementations should create such an edge only when the anchor separation clears a numerical guard,
\begin{align*}
\norm{a_i-a_j}_2>\eta_{\mathrm{anchor}},
\end{align*}
for a fixed $\eta_{\mathrm{anchor}}>0$. If a proposed new anchor is duplicate or nearly duplicate with a confused neighbor, the algorithm should skip that edge and fall back to repair, merge, or another admissible distinction.

\begin{theorem}[DGM graph generates a finite information structure]
\label{thm:dgm-graph-partition:informal}
For any finite edge set $E_t$, equality of signatures $\sigma_t^G(h)$ is an equivalence relation whose classes form a finite edge-skeleton partition $\calP_t^G$. Adding one edge $g$ refines this skeleton by $\calP_t^G\vee\sigma(g)$.
\end{theorem}

\subsection{Soft-DGM edge-gated prediction}

The hard route selects the nearest eligible anchor using fixed anchors, accepted edges, and anchor-distance order, so repairs that change only memories and centroids do not change $\calF_t^{\mathrm{DGM}}$. Given $h$, Soft-DGM instead computes retrieval distances $d_j(h)=\norm{h-c_j}_2^2$ to movable centroids. Moving centroids can change the smooth routing weights and therefore the actual Soft-DGM predictor, even when the Hard-DGM skeleton is fixed. Thus centroid repair is a repair in the formal hard-route structure, not a strict claim that the soft read map remains partition-measurable. A centroid-only top-$k$ set is not sufficient for the formal hard-route guarantee: if the hard eligible anchor never enters the candidate set, edge gates cannot recover it. We therefore use a recall-oriented candidate set
\begin{align*}
N(h)=
N_{k_c}^{\mathrm{centroid}}(h)
\cup
N_{k_a}^{\mathrm{anchor}}(h)
\cup
\operatorname{GraphNeighbors}\!\left(N_{k_c}^{\mathrm{centroid}}(h)\right),
\end{align*}
with duplicates removed and an optional budget cap after the union. The first term gives adaptive locality, the second protects the anchor-defined hard route, and the third recovers concepts connected by recent consequence conflicts. In exact small-scale implementations one may take $N(h)=\{1,\ldots,N_t\}$.
If $\operatorname{Inc}_t(j)$ is the set of edges incident to concept $j$, define the soft compatibility of $h$ with concept $j$ by
\begin{align*}
\log \chi_j(h)=
\frac{1}{\sqrt{\max\{1,\abs{\operatorname{Inc}_t(j)}\}}}
\sum_{e\in \operatorname{Inc}_t(j)}
\beta_e\log \operatorname{sigmoid}\left(\frac{s_{e,j}(u_e^\top h-b_e)}{\tau_e}\right),
\end{align*}
where structural distinction edges have $\beta_e\in(0,1]$ and the empty sum gives $\chi_j(h)=1$. The degree normalization avoids systematically suppressing high-degree concepts while preserving hard rejection in the low-temperature limit. The edge-gated routing weights are
\begin{align}
\alpha_j(h)=
\frac{\exp(-d_j(h)/T)\chi_j(h)}
{\sum_{r\in N(h)}\exp(-d_r(h)/T)\chi_r(h)},
\qquad j\in N(h).
\label{eq:edge-gated-routing}
\end{align}
The prediction is the local mixture
\begin{align*}
\hat y=\sum_{j\in N(h)}\alpha_j(h)M_j(h).
\end{align*}

The hard route makes the information-structure connection explicit. A concept $j$ is eligible for $h$ if every incident edge bit places $h$ on the side recorded for $j$:
\begin{align*}
\begin{cases}
g_e(h)=1,& s_{e,j}=+1,\\
g_e(h)=0,& s_{e,j}=-1,
\end{cases}
\qquad\text{for every }e\in \operatorname{Inc}_t(j).
\end{align*}
This convention assigns boundary points to the nonpositive side because $g_e(h)=\one\{u_e^\top h>b_e\}$. The exact hard route used in the refinement theory routes to the nearest eligible anchor, falling back to the nearest anchor if no eligible concept is available. The soft read in \eqref{eq:edge-gated-routing} is the trainable implementation: it may retrieve by movable centroids, but those centroids are retrieval statistics rather than formal information cells.

The distinction partition alone does not determine the whole exact hard route, because eligible concepts still need to be ordered. Anchor distances provide this stable order geometry. Let $\calF_t^{\mathrm{edge}}=\sigma(g_e:e\in E_t)$ and let $\calF_t^{\mathrm{anchor\text{-}order}}$ be the finite sigma-algebra generated by all pairwise anchor-distance comparison events
\begin{align*}
\{h:\norm{h-a_i}_2^2\le \norm{h-a_j}_2^2\},
\qquad 1\le i,j\le N_t,
\end{align*}
with deterministic tie-breaking. The hard routing skeleton is measurable with respect to
\begin{align*}
\calF_t^{\mathrm{DGM}}=
\calF_t^{\mathrm{edge}}\vee \calF_t^{\mathrm{anchor\text{-}order}}.
\end{align*}
The exact DGM route map can be written as
\begin{align*}
I_{\calS_t}(h)
=
\left(
(g_e(h))_{e\in E_t},
\operatorname{HardRoute}_{\calS_t}(h)
\right).
\end{align*}
This map is measurable with respect to $\calF_t^{\mathrm{DGM}}$, so
\begin{align*}
\sigma(I_{\calS_t})\subseteq \calF_t^{\mathrm{edge}}\vee \calF_t^{\mathrm{anchor\text{-}order}}.
\end{align*}
The inclusion can be strict: the full anchor-order sigma-algebra records pairwise order comparisons that may not affect the final selected concept. One may use $\calF_t^{\mathrm{DGM}}$ as a stable upper information structure with simple repair-stability proofs, or use the coarser exact-route sigma-algebra $\sigma(I_{\calS_t})$ when the selected memory location itself is the desired state. Nodes are memory locations selected by this measurable route map; they need not be in one-to-one correspondence with atoms of the edge-skeleton partition. The soft read keeps the same anchored graph skeleton but may use movable centroids and smooth gates for locality; repairs can therefore change its effective smooth read geometry. Soft-DGM should be read as a differentiable approximation to the anchor-generated hard information map, with exact partition claims recovered only in the hard-route limit or under explicit approximation assumptions.

\begin{theorem}[Anchor-based hard routing is graph-plus-anchor-order measurable]
\label{thm:hard-dgm-measurable:informal}
With finite $E_t$, finite frozen anchors $a_1,\ldots,a_{N_t}$, and deterministic tie-breaking, the exact hard-route selected concept index, defined as the nearest eligible anchor with fallback to the nearest anchor, is measurable with respect to $\calF_t^{\mathrm{DGM}}=\calF_t^{\mathrm{edge}}\vee \calF_t^{\mathrm{anchor\text{-}order}}$.
\end{theorem}

The next statement is analytic rather than structural: it applies to whichever fixed distances a soft read uses. With anchor distances it recovers the exact hard route; with centroid distances it describes the behavior of the soft retrieval approximation. It also exposes the candidate-recall requirement that links the formal hard route to the sparse implementation.

\begin{condition}[Candidate recall]
\label{cond:candidate-recall}
Let $j^\star(h)$ be the unique concept selected by the exact hard route for query $h$. The sparse soft read satisfies candidate recall at $h$ if
\begin{align*}
j^\star(h)\in N(h).
\end{align*}
If this condition fails, the soft read has no guarantee of selecting the hard route, regardless of how sharp the edge gates are.
\end{condition}

\begin{theorem}[Soft routing approaches hard routing]
\label{thm:soft-routing-hard-limit:informal}
Assume candidate recall holds for $h$, and let $j^\star$ be the recalled hard-route concept. Split the remaining candidates into eligible competitors, which satisfy the hard structural constraints, and wrong-side competitors, which violate at least one structural edge. Eligible competitors are removed by distance: if $j^\star$ has distance margin $\delta>0$ over every eligible competitor and the eligible/reference compatibility ratios are uniformly bounded, their total soft mass decays exponentially as $T\to0$. Wrong-side competitors are removed by edge gates: if every wrong-side candidate violates some incident structural edge with weight at least $\beta_{\min}>0$, signed margin at least $\delta_e>0$, and degree at most $D_{\max}$, then its compatibility decays uniformly after degree normalization. The hard-route limit is therefore an ordered or controlled joint limit: edge temperature must suppress wrong-side candidates before distance temperature makes nearest-neighbor selection decisive. Equivalently, either the wrong-side/reference distance-score ratios remain uniformly bounded, or for distance advantage $\Delta_d$ of wrong-side candidates over an eligible reference,
\begin{align*}
\frac{\beta_{\min}\delta_e}{\sqrt{D_{\max}}\tau_e}-\frac{\Delta_d}{T}\to+\infty.
\end{align*}
Under either condition, total wrong-side routing mass vanishes. Together with the distance margin for eligible competitors, this gives convergence to the hard route. Without candidate recall, the theorem gives no correctness guarantee for an omitted hard-route concept.
\end{theorem}

\begin{algorithm}[H]
\caption{\textsc{Predict} for a Distinction Graph Machine}
\label{alg:dgm-predict}
\begin{algorithmic}[1]
\State \textbf{Input:} state $\calS_t=(\{a_j,c_j,M_j,n_j\}_{j=1}^{N_t},G_t)$, encoder $\phi_\theta$, example $x$
\State $h\gets\phi_\theta(x)$
\State $N(h)\gets N_{k_c}^{\mathrm{centroid}}(h)\cup N_{k_a}^{\mathrm{anchor}}(h)\cup\operatorname{GraphNeighbors}(N_{k_c}^{\mathrm{centroid}}(h))$
\For{$j\in N(h)$}
\State compute edge compatibility $\chi_j(h)$ from incident distinctions
\State compute edge-gated weight $\alpha_j(h)$ by \eqref{eq:edge-gated-routing}
\EndFor
\State \Return $\sum_{j\in N(h)}\alpha_j(h)M_j(h)$
\end{algorithmic}
\end{algorithm}

\subsection{Refine or repair on a graph}

After observing consequence $y$, DGM scores empirical versions of the two gaps in Theorem~\ref{thm:error-decomposition}. A repair candidate keeps the DGM information map fixed and updates only local state. A refinement candidate changes the information map by adding an anchored concept and distinction edges. The committed operation is the one whose held-out gain justifies its cost and safety margin.

Let $j^\star$ be the highest-weight concept under \eqref{eq:edge-gated-routing}. Define local surprise
\begin{align*}
\rho_t=L(y,M_{j^\star}(h)).
\end{align*}
Define neighborhood consequence disagreement on $N(h)$ by the task-appropriate local prediction divergence. For classification one may use
\begin{align*}
D_t=\max_{i,r\in N(h)}\operatorname{TV}(p_i(\cdot\mid h),p_r(\cdot\mid h)),
\end{align*}
where $p_j$ is the distribution represented by memory $M_j$. For regression one may use
\begin{align*}
D_t=\max_{i,r\in N(h)}\norm{\mu_i(h)-\mu_r(h)}_2,
\end{align*}
and for language modeling the log-loss surprise $-\log p_{j^\star}(x_{t+1}\mid x_{\le t})$ is the local disagreement signal.
These quantities are proposal signals. They decide which candidates are worth scoring; they do not define repair or refinement.

All gap estimates below are computed on a held-out scoring buffer $\calB_t^{\mathrm{score}}$. Refinement proposals may use a disjoint proposal buffer $\calB_t^{\mathrm{prop}}$ together with the current observation; the scoring buffer is held out from proposal generation and from previous accept/reject decisions that determine the current state. Theorem~\ref{thm:safe-refinement:informal} applies when each scoring decision uses fresh held-out samples, or samples not previously used to choose proposals or update the state. Reservoir sampling can maintain disjoint proposal and scoring streams, but reusing the same validation buffer adaptively requires reusable-holdout or adaptive-validation control. Without that extra control, the empirical rule is a heuristic rather than the Hoeffding guarantee.

The edges created by a refinement are attached to a concrete confused-neighbor set. Let $\operatorname{Div}_{\calY}(M_i,M_r;h)$ denote a task-appropriate consequence divergence, such as total variation for classifiers, Euclidean mean distance for regression memories, or next-token distribution divergence for language models. Define
\begin{align*}
\operatorname{Confused}(h,y)
=
\left\{
i\in N(h):
L(y,M_i(h))>\tau_{\mathrm{conflict}}
\ \text{or}\
\operatorname{Div}_{\calY}(M_i,M_{j^\star};h)>\tau_{\mathrm{disagree}}
\right\}.
\end{align*}
When a budget of at most $q$ new edges is used, DGM takes $\operatorname{Confused}_q(h,y)$ to be the $q$ highest-scoring members of this set under
\begin{align*}
\operatorname{score}_i(h,y)
=
\alpha_i(h)L(y,M_i(h))
+\gamma_{\mathrm{div}}\operatorname{Div}_{\calY}(M_i,M_{j^\star};h).
\end{align*}
If the set is empty but refinement is otherwise accepted, the implementation falls back to the highest-score candidates in $N(h)\setminus\{j^\star\}$. Thus ``confused neighbors'' denotes a concrete operator rather than an informal phrase.

Repair is guarded rather than blindly committed. The implementation proposes a small set of repair candidates, such as a memory-only update, a memory-plus-centroid update, and one or more reduced centroid steps:
\begin{align*}
\calC_{\mathrm{repair}}(\calS_t,h,y)
=
\{\calS_t^{\mathrm{mem}},\calS_t^{\mathrm{mem+centroid}},\calS_t^{\mathrm{shrink}},\calS_t\}.
\end{align*}
Each candidate keeps anchors and edges fixed, so it preserves $I_{\calS_t}$ and $\calF_t^{\mathrm{DGM}}$. On the scoring buffer, the empirical repair gain is
\begin{align}
\widehat\Delta_{\mathrm{repair}}
=
\widehat R(\calS_t;\calB_t^{\mathrm{score}})
-
\min_{\calS'\in\calC_{\mathrm{repair}}}
\widehat R(\calS';\calB_t^{\mathrm{score}}).
\label{eq:repair-gain}
\end{align}
DGM's guarded repair candidate is
\begin{align}
\calS_{t+1}\in
\argmin_{\calS'\in\calC_{\mathrm{repair}}}
\widehat R(\calS';\calB_t^{\mathrm{score}})
\quad\text{subject to}\quad
\widehat R(\calS';\calB_t^{\mathrm{score}})
\le
\widehat R(\calS_t;\calB_t^{\mathrm{score}}).
\label{eq:guarded-repair}
\end{align}
The identity state is in $\calC_{\mathrm{repair}}$, so $\widehat\Delta_{\mathrm{repair}}\ge0$. Thus monotone repair is an enforced algorithmic property, not merely a theorem assumption. The anchor $a_{j^\star}$ is never moved by repair. Consequently, local memory repair and centroid adaptation do not rewrite previously accepted edge generators.

A candidate creates
\begin{align*}
a_{\mathrm{new}}=h,\qquad
c_{\mathrm{new}}=h,\qquad
M_{\mathrm{new}}=\operatorname{InitMemory}(h,y),
\end{align*}
and adds the actual accepted edge set $E_{\mathrm{new}}$ from the new anchored concept to well-separated members of $\operatorname{Confused}_q(h,y)$. Let $\widehat R(\calS;\calB_t^{\mathrm{score}})$ be the empirical local loss on the scoring buffer after routing with state $\calS$. The empirical penalized refinement gain is
\begin{align}
\widehat\Delta_{\mathrm{refine}}
=\widehat R(\calS_t;\calB_t^{\mathrm{score}})
-
\widehat R(\calS_t^{\mathrm{new}};\calB_t^{\mathrm{score}})
-
\lambda_{\mathrm{concept}}
-
\lambda_{\mathrm{edge}}\abs{E_{\mathrm{new}}}.
\label{eq:refine-gain}
\end{align}
Here $\lambda_{\mathrm{concept}}$ is the penalty for creating the anchored concept and its induced anchor-order bookkeeping, while $\lambda_{\mathrm{edge}}\abs{E_{\mathrm{new}}}$ accounts for the accepted edge generators. Using the edge budget $q$ instead of the actual $\abs{E_{\mathrm{new}}}$ is a conservative implementation choice that only makes the acceptance test stricter.
The candidate is accepted only if it beats the repair gain by a safety margin:
\begin{align}
\widehat\Delta_{\mathrm{refine}}>
\max\{\widehat\Delta_{\mathrm{repair}},0\}
+\tau+2\epsilon_m
\qquad\text{and}\qquad
D_t>\tau_{\mathrm{disagree}}.
\label{eq:refine-accept}
\end{align}
Here $\epsilon_m$ is a statistical safety margin for a scoring buffer of size $m=\abs{\calB_t^{\mathrm{score}}}$; it can be set to zero for a purely empirical implementation, or to the adaptive validation margin in Theorem~\ref{thm:safe-refinement:informal}. If the refinement test fails, DGM commits the guarded repair candidate in \eqref{eq:guarded-repair}. This makes refinement a statistically guarded gap comparison rather than an unscored growth heuristic.

\begin{algorithm}[H]
\caption{\textsc{Observe} for a Distinction Graph Machine}
\label{alg:dgm-observe}
\begin{algorithmic}[1]
\State \textbf{Input:} state $\calS_t$, labeled consequence $(x,y)$
\State $h\gets\phi_\theta(x)$
\If{$N_t=0$}
\State create anchor and centroid $a_1=c_1=h$ with memory $\operatorname{InitMemory}(h,y)$
\State \Return updated state
\EndIf
\State retrieve $N(h)$ and compute edge-gated routing weights
\State $j^\star\gets\argmax_{j\in N(h)}\alpha_j(h)$
\State $\rho\gets L(y,M_{j^\star}(h))$
\State compute neighborhood disagreement $D_t$
\State propose repair candidates with anchors and edges fixed
\State compute empirical repair gain $\widehat\Delta_{\mathrm{repair}}$ by \eqref{eq:repair-gain}
\If{$\rho>\tau_{\mathrm{repair}}$ or $D_t>\tau_{\mathrm{disagree}}$}
\State form a candidate new anchor/centroid from $h$ and $\calB_t^{\mathrm{prop}}$, with guarded candidate edges to separated members of $\operatorname{Confused}_q(h,y)$
\State estimate penalized refinement gain $\widehat\Delta_{\mathrm{refine}}$ on $\calB_t^{\mathrm{score}}$ by \eqref{eq:refine-gain}
\If{$\widehat\Delta_{\mathrm{refine}}>\max\{\widehat\Delta_{\mathrm{repair}},0\}+\tau+2\epsilon_m$ and $D_t>\tau_{\mathrm{disagree}}$}
\State commit the new anchored concept and its distinction edges
\State \Return updated state
\EndIf
\EndIf
\State commit the best candidate satisfying the guarded repair test \eqref{eq:guarded-repair}
\State \Return updated state
\end{algorithmic}
\end{algorithm}

\begin{definition}[Guarded monotone local repair]
\label{def:monotone-repair}
A guarded repair operator is monotone on scoring buffer $\calB_t^{\mathrm{score}}$ if it commits a candidate state $\calS_{t+1}$ only when local memories and retrieval centroids may change, anchors and edges remain fixed, and
\begin{align*}
\widehat R(\calS_{t+1};\calB_t^{\mathrm{score}})
\le
\widehat R(\calS_t;\calB_t^{\mathrm{score}}).
\end{align*}
The identity candidate $\calS_{t+1}=\calS_t$ is always allowed, so the guard is feasible even when every nontrivial repair would hurt the scoring buffer.
\end{definition}

\begin{theorem}[Refine-or-repair objective descent]
\label{thm:refine-repair-descent:informal}
Let the local scoring objective on a fixed scoring buffer $\calB_t^{\mathrm{score}}$ be
\begin{align*}
J_{\calB_t^{\mathrm{score}}}(\calS)
=
\widehat R(\calS;\calB_t^{\mathrm{score}})
+\lambda_{\mathrm{concept}}N_{\mathrm{concept}}(\calS)
+\lambda_{\mathrm{edge}}\abs{E(\calS)}.
\end{align*}
If repair is committed by the guarded rule \eqref{eq:guarded-repair}, then $J_{\calB_t^{\mathrm{score}}}$ does not increase. If a refinement adding one anchored concept and actual edge set $E_{\mathrm{new}}$ is accepted only under \eqref{eq:refine-accept}, then the empirical penalized objective $J_{\calB_t^{\mathrm{score}}}$ decreases by more than $\widehat\Delta_{\mathrm{repair}}+\tau+2\epsilon_m$ and in particular by more than $\tau+2\epsilon_m$.
\end{theorem}

This is a local-buffer statement. It does not imply monotone descent of a global objective when the scoring buffer changes over time; a global finiteness claim requires a common potential or validation objective.

\begin{theorem}[Statistically safe adaptive refinement]
\label{thm:safe-refinement:informal}
Assume losses lie in $[0,1]$. Let $(\mathfrak H_t)$ be the filtration containing the past stream, the current DGM state, and all proposal-buffer information available before scoring at time $t$. Let $\calC_t$ be a finite candidate class that is predictable with respect to $\mathfrak H_t$, and let $\calB_t^{\mathrm{score}}$ contain $m_t$ fresh held-out samples, or samples not previously used to choose proposals or update the state, independent of $\mathfrak H_t$ conditional on the current scoring distribution. For candidate $g$, let $\widehat G_t(g)$ be the empirical raw risk decrease on $\calB_t^{\mathrm{score}}$ before complexity cost, and let $G_t(g)$ be the corresponding population raw gain. Define
\begin{align*}
\epsilon_t=
\sqrt{
\frac{\log(4t^2(\abs{\calC_t}+1)/\delta)}
{2m_t}
}.
\end{align*}
Then, with probability at least $1-\delta$, for every time $t$ and every $g\in\calC_t$,
\begin{align*}
\abs{\widehat G_t(g)-G_t(g)}
\le
2\epsilon_t.
\end{align*}
Hence an accepted refinement satisfying
\begin{align*}
\widehat G_t(g)>
\lambda\operatorname{cost}(g)+\tau+2\epsilon_t
\end{align*}
equivalently $\widehat\Delta_t^{\mathrm{pen}}(g)=\widehat G_t(g)-\lambda\operatorname{cost}(g)>\tau+2\epsilon_t$, has positive population penalized gain after paying complexity cost and margin $\tau$, uniformly over all accepted times.
\end{theorem}

\begin{corollary}[Bounded language-model refinement score]
\label{cor:lm-clipped-refinement:informal}
For language modeling, raw next-token negative log likelihood is unbounded. Fix $L_{\max}>0$ and define the clipped normalized score
\begin{align*}
\ell_{\mathrm{clip}}(x_{\le t},x_{t+1})
=
\frac{1}{L_{\max}}
\min\{-\log p(x_{t+1}\mid x_{\le t}),L_{\max}\}.
\end{align*}
If refinement proposal and scoring use $\ell_{\mathrm{clip}}$, then Theorem~\ref{thm:safe-refinement:informal} applies to clipped-loss refinement gain. An accepted refinement has positive clipped population gain after cost and margin with the stated high probability. This corollary does not claim positive gain for the raw unbounded negative log likelihood without additional tail assumptions.
\end{corollary}

\begin{theorem}[Anchor-stable DGM information structure]
\label{thm:anchor-stable:informal}
Let
\begin{align*}
\calF_t^{\mathrm{DGM}}
=
\sigma(g_e:e\in E_t)
\vee
\sigma\!\left(\{h:\norm{h-a_i}_2^2\le \norm{h-a_j}_2^2\}:1\le i,j\le N_t\right).
\end{align*}
If a repair step changes only retrieval centroids $c_j$, local memories $M_j$, and counts $n_j$, while keeping anchors $a_j$ and edges $E_t$ fixed, then $\calF_{t+1}^{\mathrm{DGM}}=\calF_t^{\mathrm{DGM}}$. If a refinement adds a new anchor and new edge generators, then the updated information structure is the join of $\calF_t^{\mathrm{DGM}}$ with the new anchor-order events and the new edge generators.
\end{theorem}

Thus the edge skeleton realizes the minimal join operation from Section~\ref{sec:minimal-refinement}. The exact hard-route DGM adds anchor-order events as routing infrastructure whenever a new anchor is inserted. Consequently, Hard-DGM is a structured implementation of refinement, but its exact-route information update is not always the single-event minimal refinement in the strict Section~\ref{sec:minimal-refinement} sense.

\begin{corollary}[Finite number of accepted refinements]
\label{cor:finite-refinements:informal}
If there is a common nonnegative potential $\Phi(\calS)$ and every accepted refinement decreases $\Phi$ by at least $\tau>0$, then the number of accepted refinements is at most $\Phi(\calS_0)/\tau$. In particular, this applies to a fixed scoring buffer or validation objective. The local-buffer descent theorem alone should not be read as a global finiteness result when the scoring buffer changes over time.
\end{corollary}

\subsection{Data-structure operations}

\begin{definition}[DGM data-structure operations]
\label{def:dgm-data-structure-operations}
Viewed as a dynamic partition data structure, DGM implements four operations:
\begin{align*}
\operatorname{Query}(h),\qquad
\operatorname{Repair}(h,y),\qquad
\operatorname{Refine}(h,y,g),\qquad
\operatorname{Score}(g,\calB).
\end{align*}
The maintained edge-skeleton invariant is
\begin{align*}
\operatorname{Repair}\quad&\Longrightarrow\quad \calP^G(\calS_{t+1})=\calP^G(\calS_t),\\
\operatorname{Refine}(g)\quad&\Longrightarrow\quad \calP^G(\calS_{t+1})=\calP^G(\calS_t)\vee\sigma(g),
\end{align*}
where $\calP^G(\calS_t)$ is the edge-generated skeleton partition. The exact hard-route invariant is
\begin{align*}
\operatorname{Repair}\quad&\Longrightarrow\quad \calF^{\mathrm{DGM}}(\calS_{t+1})=\calF^{\mathrm{DGM}}(\calS_t),\\
\operatorname{Refine}\quad&\Longrightarrow\quad \calF^{\mathrm{DGM}}(\calS_{t+1})
=
\calF^{\mathrm{DGM}}(\calS_t)\vee\sigma(g_e:e\in E_{\mathrm{new}})\vee\sigma(\text{new anchor-order events}).
\end{align*}
Thus repair modifies payloads on the current structure, while refinement changes the structure by adding both edge generators and the order information induced by new anchors.
\end{definition}

\begin{theorem}[DGM data-structure cost]
\label{thm:dgm-data-structure-main}
Assume $N_t$ concepts in $\R^d$, candidate set size $\abs{N(h)}=k$, selected incident edge count $Q(h)$, retrieval cost $C_{\mathrm{ret}}(N_t,d)$, and selected local-memory evaluation cost $C_{\mathrm{mem}}$. Then
\begin{align*}
\operatorname{Query}(h)
=
O(C_{\mathrm{ret}}(N_t,d)+Q(h)d+kd+C_{\mathrm{mem}}).
\end{align*}
Here $Q(h)$ is an edge count and each dense linear edge gate costs $O(d)$ to evaluate. A repair adds the cost of scoring the proposed local-memory candidates on the scoring buffer. A refinement creating one anchor-centroid pair and $q$ guarded linear distinction edges adds $O(d+qd)$ state-update time, plus the indexing or comparison bookkeeping used to maintain anchor-order queries. With sparse categorical memories of average support $\bar s$, the state uses
\begin{align*}
O(N_t(2d+\bar s)+\abs{E_t}d)
\end{align*}
memory, up to indexing overhead.
\end{theorem}

The theorem is an implementation bound rather than a statistical guarantee. It says that DGM realizes the abstract operation $\Pi\mapsto\Pi\vee U$ on the edge skeleton and the corresponding join update on $\calF^{\mathrm{DGM}}$ with local graph updates and sparse routing, provided the candidate retrieval layer supplies the relevant hard-route concepts.

\subsection{Canonical local memory}

\begin{definition}[Calibrated categorical local memory]
\label{def:calibrated-categorical-memory}
The local memory $M_j$ is the repair mechanism inside an already chosen concept. For classification and language modeling, the main-text canonical memory is a calibrated categorical memory rather than an unconstrained embedding score. Concept $j$ stores token or label counts $n_{j,y}$ and $n_j=\sum_y n_{j,y}$. Given a backbone distribution $p_0(y\mid h)$ and smoothing strength $\alpha_0>0$, define
\begin{align}
p_j(y\mid h)
=
\frac{n_{j,y}+\alpha_0 p_0(y\mid h)}
{n_j+\alpha_0}.
\label{eq:concept-categorical-memory}
\end{align}
The routed memory distribution is
\begin{align}
p_{\mathrm{mem}}(y\mid h)
=
\sum_{j\in N(h)}\alpha_j(h)p_j(y\mid h),
\label{eq:routed-memory-distribution}
\end{align}
and the final prediction is the convex mixture
\begin{align}
p_{\mathrm{DGM}}(y\mid h)
=
(1-\lambda_t)p_0(y\mid h)+\lambda_t p_{\mathrm{mem}}(y\mid h),
\qquad
\lambda_t\in[0,1].
\label{eq:calibrated-dgm-mixture}
\end{align}
\end{definition}
This keeps the read path a valid probability distribution. It also gives the simple safety bound
\begin{align*}
-\log p_{\mathrm{DGM}}(y\mid h)
\le
-\log p_0(y\mid h)-\log(1-\lambda_t)
\end{align*}
whenever $\lambda_t<1$. In experiments where log loss is central, $\lambda_t$ should be selected online by a validation rule or, under bounded or clipped validation losses, by Hedge over a small grid of trust weights.

Because $p_0(y\mid h)$, $M_j(h)$, and the soft weights can depend on the hidden state, the practical predictor need not be constant on atoms of the edge partition alone. The formal interpretation is the base-information convention from Section~\ref{sec:preliminaries}: let
\begin{align*}
\calB=\sigma\!\left(p_0(\cdot\mid h),\text{fixed pre-outcome features used outside DGM routing}\right).
\end{align*}
Then DGM refinements are claims about the additional runtime information in $\calF_t^{\mathrm{DGM}}$, and predictors are measured as $\calB\vee\calF_t^{\mathrm{DGM}}$-measurable. The hidden state may be used by the algorithm to compute admissible distinctions, proposal scores, and restricted routing functions, but it is not treated as arbitrary freely conditionable information in the unrestricted Bayes-risk theory. If full $h$ is admitted into $\calB$, then all edge predicates and anchor-order events are already $\calB$-measurable and the partition-level refinement gap collapses to zero. In the hard one-hot special case, where each memory returns an atomwise constant action and no extra base features are used, this reduces to strict partition measurability; $h$-dependent soft memories should be read as restricted-class or approximation statements.

Repair is the sufficient-statistic update. For every selected concept $j\in N(h)$,
\begin{align*}
n_{j,y}^+=n_{j,y}+\alpha_j(h),
\qquad
n_j^+=n_j+\alpha_j(h),
\end{align*}
with all unselected counts unchanged. The low-cost hard-update variant is recovered by setting $\alpha_{j^\star}=1$ for the highest-weight concept and $\alpha_j=0$ for all $j\ne j^\star$. Embedding-mean memories and ridge or projection-style local repair are useful implementation variants, but they should be evaluated with calibration metrics before being used for log-loss claims.
