\section{Distinction Graph Machines}
\label{sec:algorithms}

The theory so far says what must happen when an atom is wrong: the learner must add a distinction. A runtime model then needs a data structure that can create distinctions without destroying the local state that is still useful. A tree is a useful finite special case, but it is not the most natural runtime data structure. A tree makes every decision through one rooted path. Hidden-state conflicts are often local: a new concept may need to be separated from a few confused neighbors without rewriting a global hierarchy.

\subsection{Why a graph, not a tree?}

DGM uses a graph of concepts. Local memories live at nodes, and distinctions live on edges between concepts that have been confused. The central object is
\begin{align*}
\text{DGM}=\text{graph-generated information structure}+\text{local memory repair}.
\end{align*}
This section defines that object. The graph is not metadata: its edges generate the finite information structure and gate the read path. The partition and generated-sigma-algebra language follows standard finite measurable-space constructions \citep{billingsley1995probability,kallenberg2002foundations}. The main-text statements below give the operational theorem forms; full proofs are collected in Appendix~\ref{app:algorithm-bridge-proofs}.

\subsection{State: anchors, centroids, memories, and edges}

Let $\phi_\theta:\calX\to\R^d$ be an encoder and write $h=\phi_\theta(x)$. A Distinction Graph Machine has runtime state
\begin{align*}
\calS_t=\left(\{a_j,c_j,M_j,n_j\}_{j=1}^{N_t},G_t\right),
\qquad
G_t=(V_t,E_t).
\end{align*}
Each component has a role forced by the refine-or-repair split. A runtime concept needs a stable identity, so $a_j\in\R^d$ is a frozen distinction anchor. Retrieval must adapt as more examples arrive, so $c_j\in\R^d$ is a movable retrieval centroid. State inside an adequate concept must be repairable, so $M_j$ is a local memory and $n_j$ is a usage count. If a concept is created because it was confused with existing concepts, that confusion should not disappear after the update; it becomes an edge in $E_t$. A new concept is initialized with $a_j=c_j=h$. Subsequent repair may move $c_j$, but anchors are fixed once created, so old distinctions keep their original semantics.

An edge $e=(i,j)$ stores a linear distinction
\begin{align*}
u_e=\frac{a_i-a_j}{\norm{a_i-a_j}_2},\qquad b_e=\frac{1}{2}u_e^\top(a_i+a_j),
\end{align*}
with side labels $s_{e,i}=+1$ and $s_{e,j}=-1$. Its binary generator is
\begin{align*}
g_e(h)=\one\{u_e^\top h>b_e\}.
\end{align*}

The edge set induces an information structure. Define the graph signature
\begin{align*}
\sigma_t^G(h)=\big(g_e(h)\big)_{e\in E_t}\in\{0,1\}^{E_t}.
\end{align*}
Two hidden states are indistinguishable by the current graph if their signatures agree. The nonempty level sets
\begin{align*}
A_a^t=\{h:\sigma_t^G(h)=a\},\qquad a\in\{0,1\}^{E_t},
\end{align*}
form a finite partition $\calP_t^G$ of hidden-state space, and the generated sigma-algebra is $\sigma(\calP_t^G)$. Adding a new edge is therefore a join with one binary generator:
\begin{align*}
\calP_{t+1}^G=\calP_t^G\vee\sigma(g_e).
\end{align*}
Anchors generate the distinction skeleton. Centroids and local memories are adaptive statistics attached to atoms or sparse neighborhoods of this graph-generated partition.

\begin{theorem}[Prototype distinction separates its endpoints]
\label{thm:prototype-edge:informal}
If $a_i\ne a_j$ and $u_e,b_e$ are defined as above for edge $e=(i,j)$, then
\begin{align*}
u_e^\top a_i-b_e=\frac{1}{2}\norm{a_i-a_j}_2,
\qquad
u_e^\top a_j-b_e=-\frac{1}{2}\norm{a_i-a_j}_2.
\end{align*}
Thus each edge distinction separates the two anchors that caused it.
\end{theorem}

\begin{theorem}[DGM graph generates a finite information structure]
\label{thm:dgm-graph-partition:informal}
For any finite edge set $E_t$, equality of signatures $\sigma_t^G(h)$ is an equivalence relation whose classes form a finite partition $\calP_t^G$. Adding one edge $g$ refines the partition by $\calP_t^G\vee\sigma(g)$.
\end{theorem}

\subsection{Edge-gated prediction}

Given $h$, the soft implementation first computes retrieval distances $d_j(h)=\norm{h-c_j}_2^2$ to movable centroids. A centroid-only top-$k$ set is not enough for the theory: if the hard eligible anchor never enters the candidate set, edge gates cannot recover it. We therefore use a recall-oriented candidate set
\begin{align*}
N(h)=
N_{k_c}^{\mathrm{centroid}}(h)
\cup
N_{k_a}^{\mathrm{anchor}}(h)
\cup
\operatorname{GraphNeighbors}\!\left(N_{k_c}^{\mathrm{centroid}}(h)\right),
\end{align*}
with duplicates removed and with an optional budget cap after the union. The first term gives adaptive locality, the second protects the anchor-defined hard route, and the third recovers concepts connected by recent consequence conflicts. In small exact implementations one may simply take $N(h)=\{1,\ldots,N_t\}$.
If $I_t(j)$ is the set of edges incident to prototype $j$, define the soft compatibility of $h$ with prototype $j$ by
\begin{align*}
\log \chi_j(h)=
\frac{1}{\sqrt{\max\{1,\abs{I_t(j)}\}}}
\sum_{e\in I_t(j)}
\beta_e\log \operatorname{sigmoid}\left(\frac{s_{e,j}(u_e^\top h-b_e)}{\tau_e}\right),
\end{align*}
where structural distinction edges have $\beta_e\in(0,1]$ and the empty sum gives $\chi_j(h)=1$. The degree normalization avoids systematically suppressing high-degree prototypes while preserving hard rejection in the low-temperature limit. The edge-gated routing weights are
\begin{align}
\alpha_j(h)=
\frac{\exp(-d_j(h)/T)\chi_j(h)}
{\sum_{r\in N(h)}\exp(-d_r(h)/T)\chi_r(h)},
\qquad j\in N(h).
\label{eq:edge-gated-routing}
\end{align}
The prediction is the local mixture
\begin{align*}
\hat y=\sum_{j\in N(h)}\alpha_j(h)M_j(h).
\end{align*}

The hard version makes the information-structure connection explicit. A prototype $j$ is eligible for $h$ if every incident edge bit places $h$ on $j$'s recorded side:
\begin{align*}
\begin{cases}
g_e(h)=1,& s_{e,j}=+1,\\
g_e(h)=0,& s_{e,j}=-1,
\end{cases}
\qquad\text{for every }e\in I_t(j).
\end{align*}
This convention assigns boundary points to the nonpositive side because $g_e(h)=\one\{u_e^\top h>b_e\}$. The exact Hard-DGM object used in the refinement theory routes to the nearest eligible anchor, falling back to the nearest anchor if no eligible prototype is available. Soft-DGM in \eqref{eq:edge-gated-routing} is the trainable read path: it may retrieve by movable centroids, but those centroids are retrieval statistics rather than formal information atoms.

The distinction partition alone is not the whole exact hard route. Anchor distances add a stable order geometry. Let $\calF_t^{\mathrm{edge}}=\sigma(g_e:e\in E_t)$ and let $\calF_t^{\mathrm{anchor\text{-}order}}$ be the finite sigma-algebra generated by all pairwise anchor-distance comparison events
\begin{align*}
\{h:\norm{h-a_i}_2^2\le \norm{h-a_j}_2^2\},
\qquad 1\le i,j\le N_t,
\end{align*}
with deterministic tie-breaking. The hard routing skeleton is measurable with respect to
\begin{align*}
\calF_t^{\mathrm{DGM}}=
\calF_t^{\mathrm{edge}}\vee \calF_t^{\mathrm{anchor\text{-}order}}.
\end{align*}
Thus the exact information-structure theory applies to anchor-DGM. Soft-DGM keeps the same graph skeleton but may use movable centroids and smooth gates for locality, so it should be read as a differentiable engineering relaxation of the anchor-generated information structure rather than as a new formal refinement.

\begin{theorem}[Anchor-DGM routing is graph-plus-anchor-order measurable]
\label{thm:hard-dgm-measurable:informal}
With finite $E_t$, finite frozen anchors $a_1,\ldots,a_{N_t}$, and deterministic tie-breaking, the exact Hard-DGM selected prototype index, defined as the nearest eligible anchor with fallback to the nearest anchor, is measurable with respect to $\calF_t^{\mathrm{DGM}}=\calF_t^{\mathrm{edge}}\vee \calF_t^{\mathrm{anchor\text{-}order}}$.
\end{theorem}

The next statement is analytic rather than structural: it applies to whichever fixed distances a soft read uses. With anchor distances it recovers the exact hard anchor route; with centroid distances it describes the behavior of the soft retrieval approximation. It also exposes the candidate-recall requirement that links the formal Hard-DGM route to the sparse implementation.

\begin{condition}[Candidate recall]
\label{cond:candidate-recall}
Let $j^\star(h)$ be the unique prototype selected by exact Hard-DGM for query $h$. The sparse soft read satisfies candidate recall at $h$ if
\begin{align*}
j^\star(h)\in N(h).
\end{align*}
If this condition fails, the soft read has no guarantee of selecting the hard route, regardless of how sharp the edge gates are.
\end{condition}

\begin{theorem}[Soft routing approaches hard routing]
\label{thm:soft-routing-hard-limit:informal}
Assume candidate recall holds for $h$. If the hard nearest eligible prototype is unique within $N(h)$ with distance margin $\delta>0$, then its soft routing weight tends to one exponentially as $T\to0$. If every wrong-side candidate violates some incident structural edge with weight at least $\beta_{\min}>0$, signed margin at least $\delta_e>0$, and degree at most $D_{\max}$, then each wrong-side compatibility decays uniformly after degree normalization. If at least one eligible reference candidate has compatibility bounded below and the wrong-side/reference distance-score ratios are bounded, then total wrong-side routing mass vanishes as the edge temperature $\tau_e\to0$. Without candidate recall, the theorem gives no correctness guarantee for the omitted hard-selected prototype.
\end{theorem}

\begin{algorithm}[H]
\caption{\textsc{Predict} for a Distinction Graph Machine}
\label{alg:dgm-predict}
\begin{algorithmic}[1]
\State \textbf{Input:} state $\calS_t=(\{a_j,c_j,M_j,n_j\}_{j=1}^{N_t},G_t)$, encoder $\phi_\theta$, example $x$
\State $h\gets\phi_\theta(x)$
\State $N(h)\gets N_{k_c}^{\mathrm{centroid}}(h)\cup N_{k_a}^{\mathrm{anchor}}(h)\cup\operatorname{GraphNeighbors}(N_{k_c}^{\mathrm{centroid}}(h))$
\For{$j\in N(h)$}
\State compute edge compatibility $\chi_j(h)$ from incident distinctions
\State compute edge-gated weight $\alpha_j(h)$ by \eqref{eq:edge-gated-routing}
\EndFor
\State \Return $\sum_{j\in N(h)}\alpha_j(h)M_j(h)$
\end{algorithmic}
\end{algorithm}

\subsection{Refine or repair on a graph}

After observing consequence $y$, DGM first asks whether an existing concept can absorb the evidence. Let $j^\star$ be the highest-weight prototype under \eqref{eq:edge-gated-routing}. Define local surprise
\begin{align*}
\rho_t=L(y,M_{j^\star}(h)).
\end{align*}
Define neighborhood consequence disagreement on $N(h)$ by the task-appropriate local prediction divergence. For classification one may use
\begin{align*}
D_t=\max_{i,r\in N(h)}\operatorname{TV}(p_i(\cdot\mid h),p_r(\cdot\mid h)),
\end{align*}
where $p_j$ is the distribution represented by memory $M_j$. For regression one may use
\begin{align*}
D_t=\max_{i,r\in N(h)}\norm{\mu_i(h)-\mu_r(h)}_2,
\end{align*}
and for language modeling the log-loss surprise $-\log p_{j^\star}(x_{t+1}\mid x_{\le t})$ is the local disagreement signal.

The edges created by a refinement are attached to a concrete confused-neighbor set. Let $\operatorname{Div}_{\calY}(M_i,M_r;h)$ denote a task-appropriate consequence divergence, such as total variation for classifiers, Euclidean mean distance for regression memories, or next-token distribution divergence for language models. Define
\begin{align*}
\operatorname{Confused}(h,y)
=
\left\{
i\in N(h):
L(y,M_i(h))>\tau_{\mathrm{conflict}}
\ \text{or}\
\operatorname{Div}_{\calY}(M_i,M_{j^\star};h)>\tau_{\mathrm{disagree}}
\right\}.
\end{align*}
When a budget of at most $q$ new edges is used, DGM takes $\operatorname{Confused}_q(h,y)$ to be the $q$ highest-scoring members of this set under
\begin{align*}
\operatorname{score}_i(h,y)
=
\alpha_i(h)L(y,M_i(h))
+\gamma_{\mathrm{div}}\operatorname{Div}_{\calY}(M_i,M_{j^\star};h).
\end{align*}
If the set is empty but refinement is otherwise accepted, the implementation falls back to the highest-score candidates in $N(h)\setminus\{j^\star\}$. Thus ``confused neighbors'' is an operator, not an informal phrase.

If $\rho_t$ is small, the information structure is treated as adequate and the machine tries to repair the local state. Repair is guarded rather than blindly committed. The implementation proposes a small set of repair candidates, such as a memory-only update, a memory-plus-centroid update, and one or more reduced centroid steps:
\begin{align*}
\calC_{\mathrm{repair}}(\calS_t,h,y)
=
\{\calS_t^{\mathrm{mem}},\calS_t^{\mathrm{mem+centroid}},\calS_t^{\mathrm{shrink}},\calS_t\}.
\end{align*}
Each candidate keeps anchors and edges fixed. DGM commits the best candidate whose scoring-buffer loss is not larger than the current one:
\begin{align}
\calS_{t+1}\in
\argmin_{\calS'\in\calC_{\mathrm{repair}}}
\widehat R(\calS';\calB_t^{\mathrm{score}})
\quad\text{subject to}\quad
\widehat R(\calS';\calB_t^{\mathrm{score}})
\le
\widehat R(\calS_t;\calB_t^{\mathrm{score}}).
\label{eq:guarded-repair}
\end{align}
If no nontrivial repair passes the guard, the machine either leaves the local state unchanged or enters the refinement test below. Thus monotone repair is an enforced algorithmic property, not merely a theorem assumption. The anchor $a_{j^\star}$ is never moved by repair. Consequently, local memory repair and centroid adaptation do not rewrite previously accepted edge generators.
If local surprise and disagreement are large, DGM evaluates a candidate refinement. To match the statistical guarantee, the local evidence is logically split into a proposal buffer and a scoring buffer,
\begin{align*}
\calB_t^{\mathrm{prop}}
\qquad\text{and}\qquad
\calB_t^{\mathrm{score}}.
\end{align*}
The proposal buffer, together with the current observation, may be used to propose a new concept and confused-neighbor edges. The scoring buffer is held out from proposal generation and is used only to estimate the refinement gain. A practical implementation can maintain the split by reservoir sampling incoming observations into the two buffers. If an implementation reuses one buffer for both proposal and scoring, the empirical rule still defines a heuristic, but Theorem~\ref{thm:safe-refinement:informal} applies to the sample-split version.

A candidate creates
\begin{align*}
a_{\mathrm{new}}=h,\qquad
c_{\mathrm{new}}=h,\qquad
M_{\mathrm{new}}=\operatorname{InitMemory}(h,y),
\end{align*}
and adds distinction edges from the new anchored concept to $\operatorname{Confused}_q(h,y)$. Let $\widehat R(\calS;\calB_t^{\mathrm{score}})$ be the empirical local loss on the scoring buffer after routing with state $\calS$. The refinement gain is
\begin{align*}
\Delta_{\mathrm{refine}}
=\widehat R(\calS_t;\calB_t^{\mathrm{score}})
-\widehat R(\calS_t^{\mathrm{new}};\calB_t^{\mathrm{score}}).
\end{align*}
The candidate is accepted only if
\begin{align}
\Delta_{\mathrm{refine}}>
\lambda_{\mathrm{proto}}+\lambda_{\mathrm{edge}}q+\tau+2\epsilon_m
\qquad\text{and}\qquad
D_t>\tau_{\mathrm{disagree}}.
\label{eq:refine-accept}
\end{align}
Here $\epsilon_m$ is a statistical safety margin for a scoring buffer of size $m=\abs{\calB_t^{\mathrm{score}}}$; it can be set to zero for a purely empirical implementation, or to the finite-class bound in Theorem~\ref{thm:safe-refinement:informal}. Otherwise DGM attempts guarded local repair under the current graph. This makes refinement a statistically guarded penalized empirical-risk decision rather than an unscored heuristic.

\begin{algorithm}[H]
\caption{\textsc{Observe} for a Distinction Graph Machine}
\label{alg:dgm-observe}
\begin{algorithmic}[1]
\State \textbf{Input:} state $\calS_t$, labeled consequence $(x,y)$
\State $h\gets\phi_\theta(x)$
\If{$N_t=0$}
\State create anchor and centroid $a_1=c_1=h$ with memory $\operatorname{InitMemory}(h,y)$
\State \Return updated state
\EndIf
\State retrieve $N(h)$ and compute edge-gated routing weights
\State $j^\star\gets\argmax_{j\in N(h)}\alpha_j(h)$
\State $\rho\gets L(y,M_{j^\star}(h))$
\State compute neighborhood disagreement $D_t$
\If{$\rho\le\tau_{\mathrm{repair}}$}
\State propose repair candidates with anchors and edges fixed
\State commit the best candidate satisfying the guarded repair test \eqref{eq:guarded-repair}; otherwise leave state unchanged or enter refinement test
\Else
\State form a candidate new anchor/centroid from $h$ and $\calB_t^{\mathrm{prop}}$, with candidate edges to $\operatorname{Confused}_q(h,y)$
\State estimate $\Delta_{\mathrm{refine}}$ on the held-out scoring buffer $\calB_t^{\mathrm{score}}$
\If{$\Delta_{\mathrm{refine}}>\lambda_{\mathrm{proto}}+\lambda_{\mathrm{edge}}q+\tau+2\epsilon_m$ and $D_t>\tau_{\mathrm{disagree}}$}
\State commit the new anchored concept and its distinction edges
\Else
\State attempt guarded repair under the current graph
\EndIf
\EndIf
\State \Return updated state
\end{algorithmic}
\end{algorithm}

\begin{definition}[Guarded monotone local repair]
\label{def:monotone-repair}
A guarded repair operator is monotone on scoring buffer $\calB_t^{\mathrm{score}}$ if it commits a candidate state $\calS_{t+1}$ only when local memories and retrieval centroids may change, anchors and edges remain fixed, and
\begin{align*}
\widehat R(\calS_{t+1};\calB_t^{\mathrm{score}})
\le
\widehat R(\calS_t;\calB_t^{\mathrm{score}}).
\end{align*}
The identity candidate $\calS_{t+1}=\calS_t$ is always allowed, so the guard is feasible even when every nontrivial repair would hurt the scoring buffer.
\end{definition}

\begin{theorem}[Refine-or-repair objective descent]
\label{thm:refine-repair-descent:informal}
Let the local scoring objective on a fixed scoring buffer $\calB_t^{\mathrm{score}}$ be
\begin{align*}
J_{\calB_t^{\mathrm{score}}}(\calS)
=
\widehat R(\calS;\calB_t^{\mathrm{score}})
+\lambda_{\mathrm{proto}}N_{\mathrm{proto}}(\calS)
+\lambda_{\mathrm{edge}}\abs{E(\calS)}.
\end{align*}
If repair is committed by the guarded rule \eqref{eq:guarded-repair}, then $J_{\calB_t^{\mathrm{score}}}$ does not increase. If a refinement adding one anchored concept and $q$ edges is accepted only under \eqref{eq:refine-accept}, then the empirical penalized objective $J_{\calB_t^{\mathrm{score}}}$ decreases by more than $\tau+2\epsilon_m$.
\end{theorem}

This is a local-buffer statement. It does not imply monotone descent of a global objective when the scoring buffer changes over time; a global finiteness claim requires a common potential or validation objective.

\begin{theorem}[Statistically safe finite-class refinement]
\label{thm:safe-refinement:informal}
Assume losses lie in $[0,1]$, the scoring buffer has $m$ independent samples, and the finite candidate refinement class $\calH$ is fixed before scoring, or is generated from $\calB_t^{\mathrm{prop}}$ and the current observation, independent of $\calB_t^{\mathrm{score}}$. With probability at least $1-\delta$, empirical and population risks are uniformly within
\begin{align*}
\epsilon_m=\sqrt{\frac{\log(2(\abs{\calH}+1)/\delta)}{2m}}.
\end{align*}
Consequently every empirical gain $\widehat\Delta(g)$ differs from its population gain $\Delta(g)$ by at most $2\epsilon_m$. Hence an accepted refinement satisfying
\begin{align*}
\widehat\Delta(g)>
\lambda\operatorname{cost}(g)+\tau+2\epsilon_m
\end{align*}
has positive population gain after paying complexity cost and margin $\tau$.
\end{theorem}

\begin{corollary}[Bounded language-model refinement score]
\label{cor:lm-clipped-refinement:informal}
For language modeling, raw next-token negative log likelihood is unbounded. Fix $L_{\max}>0$ and define the clipped normalized score
\begin{align*}
\ell_{\mathrm{clip}}(x_{\le t},x_{t+1})
=
\frac{1}{L_{\max}}
\min\{-\log p(x_{t+1}\mid x_{\le t}),L_{\max}\}.
\end{align*}
If refinement proposal and scoring use $\ell_{\mathrm{clip}}$, then Theorem~\ref{thm:safe-refinement:informal} applies to clipped-loss refinement gain. An accepted refinement has positive clipped population gain after cost and margin with the stated high probability. This corollary does not claim positive gain for the raw unbounded negative log likelihood without additional tail assumptions.
\end{corollary}

\begin{theorem}[Anchor-stable DGM information structure]
\label{thm:anchor-stable:informal}
Let
\begin{align*}
\calF_t^{\mathrm{DGM}}
=
\sigma(g_e:e\in E_t)
\vee
\sigma\!\left(\{h:\norm{h-a_i}_2^2\le \norm{h-a_j}_2^2\}:1\le i,j\le N_t\right).
\end{align*}
If a repair step changes only retrieval centroids $c_j$, local memories $M_j$, and counts $n_j$, while keeping anchors $a_j$ and edges $E_t$ fixed, then $\calF_{t+1}^{\mathrm{DGM}}=\calF_t^{\mathrm{DGM}}$. If a refinement adds a new anchor and new edge generators, then the updated information structure is the join of $\calF_t^{\mathrm{DGM}}$ with the new anchor-order events and the new edge generators.
\end{theorem}

\begin{corollary}[Finite number of accepted refinements]
\label{cor:finite-refinements:informal}
If there is a common nonnegative potential $\Phi(\calS)$ and every accepted refinement decreases $\Phi$ by at least $\tau>0$, then the number of accepted refinements is at most $\Phi(\calS_0)/\tau$. In particular, this applies to a fixed scoring buffer or validation objective. The local-buffer descent theorem alone should not be read as a global finiteness result when the scoring buffer changes over time.
\end{corollary}

\subsection{Trees as a hard-routing special case}

A tree stores distinctions as a hierarchy. A mistake near the root changes every downstream routing decision. A graph stores distinctions locally between concepts. Creating a new concept does not rewrite old paths; it adds a local alternative and edges explaining why the new concept differs from its neighbors. Trees are still contained in the view below, but as a constrained hard-routing case.

\begin{proposition}[Trees as generalized hard-DGMs]
\label{prop:tree-as-dgm:informal}
Given a binary distinction tree, construct one generalized hard-DGM prototype for each leaf and attach the path distinctions as hard eligibility constraints for that prototype. If generalized hard-DGM uses exact hard routing to the unique eligible leaf prototype and each prototype memory equals the tree leaf predictor, then generalized hard-DGM and the tree make identical predictions.
\end{proposition}

Thus the distinction-graph view contains trees after allowing arbitrary prototype-level eligibility constraints. A decision tree is the sparse case in which distinctions are forced into a rooted hierarchy and routing is hard and single-path.

\subsection{Canonical local memories}

The local memory $M_j$ is the repair mechanism inside an already chosen concept. To keep the main method fixed, we use two canonical variants.

\paragraph{DGM-Mean.} For classification or language modeling with output embeddings $E[y]\in\R^{d_e}$, prototype $j$ stores
\begin{align*}
r_j=\frac{1}{n_j}\sum_{s\in C_j}E[y_s].
\end{align*}
It contributes the score
\begin{align*}
\ell_{\mathrm{DGM}}(y\mid h)=\sum_{j\in N(h)}\alpha_j(h)E[y]^\top r_j.
\end{align*}
Repair is the sufficient-statistic update, applied for every selected $j\in N(h)$ with all unselected memories unchanged:
\begin{align*}
r_j^+=\frac{n_jr_j+\alpha_j(h)E[y]}{n_j+\alpha_j(h)},
\qquad
n_j^+=n_j+\alpha_j(h).
\end{align*}
The low-cost hard-update variant is recovered by setting $\alpha_{j^\star}=1$ for the highest-weight prototype and $\alpha_j=0$ for all $j\ne j^\star$.

\paragraph{DGM-Ridge.} For few-shot adaptation or local readout repair, a memory stores a small support set $Z\in\R^{n\times d}$ and labels $Y\in\R^{n\times C}$. The closed-form adapter is the dual ridge/Tikhonov solution \citep{tikhonov1977solutions,hoerl1970ridge}:
\begin{align*}
\Gamma=(ZZ^\top+\lambda I)^{-1}Y,
\qquad
\operatorname{logits}(z)=zZ^\top\Gamma.
\end{align*}
This is the canonical stronger adapter version. Projection Memory and CPM are advanced repair rules inside $M_j$ after the graph has fixed the relevant local concept; Section~\ref{sec:projection-bridge} treats that relation directly.

\subsection{From DGM to DGM-Embedding}
\label{sec:language-dgm}

The generic DGM object above can be attached to many local memories. For sequence models, the canonical instantiation is DGM-Embedding: hidden-state concepts store next-token embedding memories, the read path produces a sparse logit correction, and the observe rule runs after the true next token is revealed. Section~\ref{sec:large-model-training} gives this layer directly, including the read-before-observe protocol and the forward-only detached write path.

\subsection{Complexity}

Let
\begin{align*}
Q(h)=\sum_{j\in N(h)}\abs{I_t(j)}
\end{align*}
be the number of incident edge gates evaluated for the selected candidates. With exact nearest-neighbor search, prediction costs
\begin{align*}
O(N_td+Q(h)+\abs{N(h)}d+C_{\mathrm{local}}),
\end{align*}
where $C_{\mathrm{local}}$ is the cost of evaluating the selected local memories. With approximate nearest-neighbor indexing, the $N_td$ term can be replaced by the query cost of the index \citep{indyk1998approximate,johnson2019billion}. Creating an anchor-centroid pair costs $O(d)$ plus the cost of initializing its memory. Adding $q$ distinction edges costs $O(qd)$. If each prototype stores an embedding mean in $\R^{d_e}$, runtime memory is
\begin{align*}
O(N_t(2d+d_e)+|E_t|d).
\end{align*}

\begin{theorem}[DGM data-structure bound]
\label{thm:dpm-complexity:informal}
For exact search over $N_t$ centroids in $\R^d$, one edge-gated DGM prediction costs $O(N_td+Q+\abs{N}d+C_{\mathrm{local}})$, where $N=N(h)$ and $Q=Q(h)$. One observation that repairs a selected prototype costs this plus the guarded local repair scoring cost. One observation that creates a new anchor-centroid pair and $q$ distinction edges costs an additional $O(d+qd)$. With embedding-mean memories, the runtime state uses $O(N_t(2d+d_e)+|E_t|d)$ memory.
\end{theorem}
