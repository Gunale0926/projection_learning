\section{Computational Checks}
\label{sec:experiments}

The theory above is structural, but the model is meant to be executable. We include small reproducible comparisons against backpropagation. The accompanying script is in the experiment directory and uses PyTorch for the backpropagation baselines and NumPy for the distinction-memory methods.

The first task is a two-context XOR stream. In the first context the label is XOR of two Gaussian cluster signs; in the second context the labels are flipped. The context coordinate is observable, but the learner must discover the relevant distinction. We compare three learners. The offline MLP is trained by ordinary backpropagation on the full stream for 500 epochs and should be read as a batch upper bound, not as a runtime learner. The online MLP predicts each example and then performs one backpropagation step on that example. The DGM-style distinction memory uses the same online protocol: predict first, then observe and refine-or-repair. Results are averaged over five seeds with 100 examples per context.

\centering

\toprule
Model & Runtime update & Prequential accuracy & Test accuracy \\
\midrule
Backprop MLP, offline & batch backprop & $-$ & $0.993\pm0.002$ \\
Backprop MLP, online & one backprop step & $0.569\pm0.065$ & $0.654\pm0.038$ \\
DGM-style memory, online & refine-or-repair & $0.804\pm0.025$ & $0.944\pm0.026$ \\
\bottomrule
\end{tabular}
\caption{DGM-style distinction memory versus backpropagation on a limited online contextual-XOR stream. Offline backprop is a batch upper bound. The relevant runtime comparison is between online backpropagation and online distinction refinement. The distinction-memory run has monotone objective decrease at accepted refinements in all runs.}
\label{tab:dgm-check}
\end{table}

The online backpropagation baseline sees each example once and performs one gradient step. Offline backpropagation is again a batch upper bound.

These experiments test a specific thesis rather than a broad benchmark claim. When the representation lacks a useful distinction, one gradient step on the current example is a weak runtime update. A distinction-memory system can instead create or retrieve a local concept and then repair local state. With full offline access and many epochs, backpropagation remains strong, as expected.

For a stronger adapter benchmark, we train a shared convolutional encoder and freeze it. The adaptation problem is then deliberately narrow: given only five labeled support examples per class, adapt the readout. We compare DGM-Ridge, a closed-form projection adapter from Section~\ref{sec:large-model-training}, against a one-pass backpropagation head and a 200-step backpropagation head on the same frozen embeddings. This isolates the large-model use case: a strong backbone already exists, and the question is whether runtime adaptation should be a gradient loop or a closed-form memory repair.

Finally, we include a synthetic language-modeling stream. Each document context induces a different next-token transition map. The learner observes a prefix representation and must predict the next token before seeing it. After the next token is revealed, DGM-style memory stores the prefix type and next-token consequence; online backpropagation again receives one gradient step per token.

We also verify the projection-memory residual contraction identity used by local repair. For random $W,k,v$ and step size $\eta$, the soft Projection Memory update satisfies
\begin{align*}
v-W_{t+1}k=\frac{1}{1+\eta\norm{k}_2^2}(v-W_tk).
\end{align*}
Over 200 random trials, the maximum absolute error between the measured residual ratio and the formula was $3.33\cdot 10^{-16}$.
