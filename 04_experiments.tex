\section{Experiments: When Refinement Matters}
\label{sec:experiments}

The experiments test the mechanism predicted by the theory rather than broad benchmark performance. A diagnostic task should contain both a repair component, where local memory updates help, and a refinement component, where repair-only methods collapse incompatible consequences into one state. All learners use a prequential protocol: predict first, observe the consequence, then update. Implementation details and additional sanity checks are in Appendix~\ref{app:additional-checks}.

\subsection{Hidden-regime sequence prediction}

The most direct check is a hidden-regime online sequence task. A hidden regime chooses the next-token transition map. The learner is not given the regime label; it observes only a noisy continuous cue and the current token, predicts the next token, and then receives the next-token consequence. The same current token can therefore require different next tokens in different regimes. A repair-only memory averages those consequences, while a refine-and-repair learner can create separate runtime concepts.

Held-out accuracies for the same runs were $0.285\pm0.049$ for cache LM, $0.346\pm0.106$ for budgeted $k$NN-LM, $0.235\pm0.024$ for nearest-centroid memory, $0.055\pm0.015$ for repair-only DGM, $0.680\pm0.065$ for refine+repair DGM, and $0.282\pm0.086$ for online backpropagation. The important comparison is structural: repair-only DGM fails because the task is constructed so that one local memory is the wrong information structure.

\subsection{Repair-only ablation}

We also run a hard-routing special case on XOR Gaussian clusters. The repair-only model has one leaf and cannot represent XOR, while refine+repair creates accepted distinctions with objective decrease. This is the smallest test of the obstruction theorem: if incompatible labels lie inside one information cell, local repair has no representation in which to store the answer.

\subsection{Controlled contextual XOR}

As a graph-memory check, we use a two-context XOR stream. In the first context the label is XOR of two Gaussian cluster signs; in the second context the labels are flipped. The context coordinate is observable, but the learner must discover how it changes the consequence. The offline MLP is a batch upper bound; the online MLP predicts each example and then performs one backpropagation step; DGM uses the same online protocol but may refine the concept graph before repairing local state.

\subsection{Synthetic next-token stream}

Finally, we include a synthetic language-modeling stream. Each document context induces a different next-token transition map. The learner observes a prefix representation and must predict the next token before seeing it. After the next token is revealed, DGM memory stores the prefix type and next-token consequence; online backpropagation again receives one gradient step per token.

These experiments do not claim that DGM is a finished competitive language-modeling system. They check the mechanism predicted by the theory: when error is caused by a missing distinction, methods that only repair state are forced to mix incompatible consequences; a refine-and-repair learner can create the missing runtime concept and then repair it locally.
