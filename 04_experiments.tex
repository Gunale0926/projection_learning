\section{Experiments and Computational Checks}
\label{sec:experiments}

The theory above is structural, but the model is meant to be executable. We include small reproducible checks organized around the paper's main claim: repair-only should fail when the stream hides a missing distinction, refine+repair should recover, edge-gated graph memory should help when local conflicts recur, and closed-form repair should be useful once the relevant concept is fixed. These checks are not meant to settle broad benchmark performance. They are designed to test whether the theorem-level separation has an observable algorithmic footprint. Stronger empirical claims would require a broader suite with online trees, LVQ, RLS, passive-aggressive updates, replay-buffer SGD, TTT-style adaptation, fast-weight or linear-attention memory, and stronger frozen-encoder baselines. The accompanying script is in the experiment directory and uses PyTorch for the backpropagation baselines and NumPy for the DGM memory methods.

The first task is a two-context XOR stream. In the first context the label is XOR of two Gaussian cluster signs; in the second context the labels are flipped. The context coordinate is observable, but the learner must discover the relevant distinction. We compare three learners. The offline MLP is trained by ordinary backpropagation on the full stream for 500 epochs and should be read as a batch upper bound, not as a runtime learner. The online MLP predicts each example and then performs one backpropagation step on that example. The DGM graph memory uses the same online protocol: predict first, then observe and refine-or-repair. Results are averaged over five seeds with 100 examples per context.

The most direct check of the runtime-learning claim is a hidden-regime online sequence task. A hidden regime chooses the next-token transition map. The learner is not given the regime label; it observes only a noisy continuous cue and the current token, predicts the next token, and then receives the next-token consequence. This setting separates cache-like repair from missing-distinction refinement: the same current token can require different next tokens in different regimes.

These experiments test a specific thesis rather than a broad benchmark claim. When the representation lacks a useful distinction, one gradient step on the current example is a weak runtime update. A DGM memory system can instead create or retrieve a local concept and then repair local state. With full offline access and many epochs, backpropagation remains strong, as expected. The image-stream results should mainly be read as implementation checks; the more relevant settings are context switching, few-shot adaptation with a frozen encoder, long-document memory, repeated syntax regimes, and hidden task identity.

For a stronger adapter benchmark, we train a shared convolutional encoder and freeze it. The adaptation problem is then deliberately narrow: given only five labeled support examples per class, adapt the readout. We compare nearest-neighbor memory, ridge memory as a closed-form projection adapter from Section~\ref{sec:large-model-training}, a one-pass backpropagation head, and a 200-step backpropagation head on the same frozen embeddings. This isolates the large-model use case: a strong backbone already exists, and the question is whether runtime adaptation should be a gradient loop, a nonparametric memory, or a closed-form local repair.

We also verify the projection-memory residual contraction identity used by local repair. For random $W,k,v$ and step size $\eta$, the soft Projection Memory update satisfies
\begin{align*}
v-W_{t+1}k=\frac{1}{1+\eta\norm{k}_2^2}(v-W_tk).
\end{align*}
Over 200 random trials, the maximum absolute error between the measured residual ratio and the formula was $3.33\cdot 10^{-16}$.
