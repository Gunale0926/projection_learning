\section{Theory-Matching Experiments}
\label{sec:experiments}

The experiments have two roles. The controlled diagnostics test the formal claims: each stream is designed so that the Bayes-optimal repair inside an existing atom is different from a refinement that changes the information structure, or so that untrusted memory can be separated from calibrated log-loss aggregation. The main structure experiment uses the CIFAR-100 hierarchy as a real semantic graph: the learner sees image-indexed fine-class atoms and must recover which coarse query is consequence-relevant. We then report explicit runtime baselines against online backpropagation and frozen-head adaptation. All online learners use a prequential protocol: predict first, observe the consequence, then update \citep{dawid1984prequential}. The implementation scripts live in \path{experiments/}; the vector figures are generated by \path{experiments/make_publication_figures.py} from the saved JSON and CSV artifacts.

\subsection{Known Bayes-gap calibration}

The first experiment tests the numerical prediction of the repair-only lower bound. We sample
\[
Z\sim \mathrm{Unif}\{1,\ldots,K\},\qquad U\sim \mathrm{Bernoulli}(1/2),
\]
and the old partition observes only $Z$, while the refined partition observes $(Z,U)$. Consequences follow
\[
Y\mid Z=z,U=u\sim \mathrm{Bernoulli}(p_{z,u}),\qquad
p_{z,0}=0.5-\delta,\quad p_{z,1}=0.5+\delta .
\]
Thus $P(Y=1\mid Z=z)=0.5$, so the old atom has log-risk $\log 2$, while the refined risk is $H(0.5+\delta)$. The predicted slope of repair-only cumulative regret is therefore
\[
\Delta_{\mathrm{theory}}=\log 2-H(0.5+\delta)=I(Y;U\mid Z).
\]
We compare smoothed count predictors for the old and refined partitions, plus a one-bit DGM count learner whose candidate distinction is either the true bit $U$ or an independent noise bit.

\subsection{CIFAR-100 hierarchy query stream}
\label{sec:hqs-experiment}

The main experiment isolates the value of a learned information structure on a non-synthetic hierarchy. CIFAR-100 supplies 100 fine labels and 20 coarse labels. At each online round we sample an image with fine class $z$ and coarse class $c(z)$, then ask a binary query $q$: ``does this image belong to coarse class $q$?'' The query is balanced,
\[
q=c(z)\text{ with probability }1/2,\qquad
q\sim\mathrm{Unif}(\{1,\ldots,20\}\setminus\{c(z)\})\text{ otherwise},
\]
and we flip the answer with label noise $\eta=0.02$. Thus $P(Y=1\mid z)=1/2$ inside every fine-class atom. A repair-only learner that conditions only on $z$ has Bayes log-risk $\log 2$, while the refined atom $q=c(z)$ versus $q\ne c(z)$ has risk $H_{\mathrm{bin}}(\eta)=0.098$.

We use the oracle-partition version: the online state receives the true fine label $z$ as its current atom. This deliberately removes visual recognition from the comparison and tests only whether the learner can discover the consequence-relevant query distinction. DGM-HQS keeps a repairable Bernoulli memory at each fine node and scores candidate edges $g_k(q)=\mathbbm{1}\{q=k\}$ using disjoint proposal and scoring buffers of size 64, with 20 independent spurious Bernoulli candidates in the pool. An edge is accepted only when its held-out log-loss gain exceeds the edge penalty and threshold. Once accepted, the node uses one positive branch $q=k$ and one pooled negative branch $q\ne k$.

The result is not that DGM beats every possible learner; crossed interaction features also succeed. The point is structural efficiency. DGM accepts one edge per fine class and pools the 19 negative coarse queries, whereas cross-product memories and crossed logistic features allocate one cell or parameter for every fine-query pair. Under the same 200-object budget, the cache baseline cannot compress the rule that all non-parent coarse queries share the same negative consequence.

\subsection{Guarded refine-or-repair decisions}

The second experiment tests whether refinement is consequence-sensitive rather than novelty-sensitive. We create two old atoms. In the repair atom, $Y\sim\mathrm{Bernoulli}(0.8)$ and all candidate distinctions are independent of $Y$, but the current local prediction is deliberately stale at $q(Y=1)=0.2$. The correct action is repair. In the refinement atom, $Y\mid U=0\sim\mathrm{Bernoulli}(0.1)$ and $Y\mid U=1\sim\mathrm{Bernoulli}(0.9)$, so the correct action is to refine by $U$.

The candidate pool contains the true distinction $U$, twenty independent noise bits, and six geometry-only bits that are visually separable but independent of $Y$. Candidate proposal and validation use separate buffers. A refinement is accepted only when its penalized held-out gain exceeds both zero and the held-out repair gain by a margin.

\subsection{Calibrated log-loss trust aggregation}

The calibration diagnostic isolates the calibration issue. We run a controlled next-token stream with repeated contexts and a phase drift. The base predictor is the phase-level unigram law, while the DGM memory routes by context and stores token counts. An unsmoothed, overconfident memory improves accuracy but can assign catastrophic probability to unseen post-drift tokens; a smoothed memory is safer, and the trust aggregator mixes a grid $\Lambda=\{0,0.05,0.1,0.2,0.4,0.7,1\}$ containing the frozen base predictor.

The generated artifact \path{experiments/calibrated_dgm_results/calibration.csv} logs per-step NLL, base NLL, Brier score, ECE, cumulative no-harm slack, and all trust weights \citep{brier1950verification,guo2017calibration,vovk1990aggregating,freund1997decision}. In the 4000-step run used as a smoke check, the base average NLL is $3.865$, the unsmoothed overconfident memory average NLL is $5.173$, the smoothed memory-only average NLL is $2.815$, and Hedge trust aggregation obtains $2.763$. The cumulative no-harm quantity
\[
\sum_{t\le T}-\log p_t(y_t)-\sum_{t\le T}-\log p_0(y_t\mid h_t)-\log|\Lambda|
\]
has maximum value $-1.719$ over the run, so the logged trajectory satisfies the theorem's fallback bound with numerical slack.

\subsection{Graph-memory ablation on contextual XOR}

The contextual XOR experiment checks the data-structure claim. The stream is a contextual XOR problem: $Y=b_1\oplus b_2\oplus c$, where the initial atoms are keyed by the two visible signs $(b_1,b_2)$ and the useful edge-generated distinction is the context bit $c$. We add nuisance coordinates so that prototype growth and cache baselines can create geometrically distinct states without obtaining the consequence-relevant partition.

\subsection{Runtime comparison against backpropagation baselines}

The relevant backpropagation baseline depends on the runtime contract. Full offline backpropagation is a batch upper bound: it can revisit all examples and change the representation quotient. The direct runtime baseline is online backpropagation, which sees each example once and makes one gradient update after the prequential prediction \citep{rumelhart1986learning,zinkevich2003online,shalev2012online,dawid1984prequential}. Figure~\ref{fig:dgm-vs-backprop} reports both, so the comparison boundary is explicit.

\begin{figure}[!htbp]
\centering
\includegraphics[width=\linewidth]{experiments/publication_figures/dgm_vs_backprop.pdf}
\caption{Runtime baselines against backpropagation. (A) On contextual XOR and synthetic sequence-map streams, DGM memory has lower prequential NLL than one-step online backpropagation. (B) DGM also has higher test accuracy; offline backpropagation is shown as a batch upper bound. (C) On frozen embeddings with five support examples per class, closed-form ridge memory obtains higher accuracy than a 200-step backpropagation head at lower adapter time. Circles denote MNIST and triangles denote CIFAR-10.}
\label{fig:dgm-vs-backprop}
\end{figure}

On the two-context XOR stream, online backpropagation obtains $0.569\pm0.065$ prequential accuracy, $0.689\pm0.019$ prequential NLL, and $0.654\pm0.038$ test accuracy. DGM obtains $0.804\pm0.025$ prequential accuracy, $0.458\pm0.038$ prequential NLL, and $0.944\pm0.026$ test accuracy. On the synthetic sequence-map stream, online backpropagation obtains $0.113\pm0.029$ prequential accuracy, $3.065\pm0.061$ prequential NLL, and $0.125\pm0.021$ test accuracy, while DGM obtains $0.862\pm0.005$, $2.067\pm0.082$, and $0.912\pm0.077$, respectively. In the frozen-embedding adapter check, ridge memory reaches $0.623\pm0.003$ MNIST accuracy in $0.0018\,\mathrm{s}$ and $0.397\pm0.010$ CIFAR-10 accuracy in $0.0004\,\mathrm{s}$, compared with $0.332\pm0.009$ and $0.359\pm0.009$ for a 200-step backpropagation head taking about $0.032\,\mathrm{s}$.

\paragraph{Scope.}
These experiments do not claim that DGM is a finished competitive language-modeling system or that runtime memory dominates unconstrained offline training. They test the mechanisms required by the theory and show the intended runtime advantage: repair-only regret has the predicted information-structure slope; the refinement rule accepts only consequence-useful distinctions; graph-generated refinement plus local repair is more effective than either operation alone; and under one-pass or low-wall-clock adaptation budgets, DGM obtains lower loss or higher accuracy than the corresponding backpropagation baseline.
