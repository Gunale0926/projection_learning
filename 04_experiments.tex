\section{Experiments}
\label{sec:experiments}

The experiments have two roles. The controlled diagnostics test the formal
claims: each stream is designed so that Bayes-optimal repair inside an existing
atom is different from a refinement that changes the information structure, or
so that untrusted memory can be separated from calibrated log-loss aggregation.
The CIFAR hierarchy experiments then test the same distinction on real image
semantics. All online learners use a prequential protocol: predict first,
observe the consequence, then update \citep{dawid1984prequential}. The
implementation scripts live in \path{experiments/}; vector figures are generated
from saved JSON and CSV artifacts.

\subsection{Known Bayes-gap calibration}

The first experiment tests the numerical prediction of the repair-only lower
bound. We sample
\[
Z\sim \mathrm{Unif}\{1,\ldots,K\},\qquad U\sim \mathrm{Bernoulli}(1/2),
\]
and the old partition observes only $Z$, while the refined partition observes
$(Z,U)$. Consequences follow
\[
Y\mid Z=z,U=u\sim \mathrm{Bernoulli}(p_{z,u}),\qquad
p_{z,0}=0.5-\delta,\quad p_{z,1}=0.5+\delta .
\]
Thus $P(Y=1\mid Z=z)=0.5$, so the old atom has log-risk $\log 2$, while the
refined risk is $H(0.5+\delta)$. The predicted slope of repair-only cumulative
regret is
\[
\Delta_{\mathrm{theory}}=\log 2-H(0.5+\delta)=I(Y;U\mid Z).
\]
We compare smoothed count predictors for the old and refined partitions, plus a
one-bit DGM count learner whose candidate distinction is either the true bit
$U$ or an independent noise bit.

\subsection{Oracle-partition CIFAR-100 BalancedMask-HQS}
\label{sec:hqs-experiment}

The oracle-partition CIFAR-100 hierarchy query stream isolates graph recovery
from visual recognition. CIFAR-100 supplies 100 fine labels and 20 coarse labels
\citep{krizhevsky2009learning}; the online state receives the true fine label
$z$ as its current atom and must discover which coarse coordinate is
consequence-relevant.

The intuitive single-query version asks whether the image belongs to one coarse
class $q$, with
\[
q=c(z)\text{ with probability }1/2,\qquad
q\sim\mathrm{Unif}(\{1,\ldots,20\}\setminus\{c(z)\})\text{ otherwise},
\]
but this sampler makes the parent query marginally frequent. We therefore use a
frequency-controlled mask-query version as the main diagnostic. In
BalancedMask-HQS, each round samples $s\in\{0,1\}^{20}$ with iid
$s_k\sim\mathrm{Bernoulli}(1/2)$ and sets
\[
Y=s_{c(z)}
\]
before flipping the label with probability $\eta=0.02$. Thus
$P(Y=1\mid z)=1/2$, so a repair-only learner that conditions only on $z$ has
Bayes log-risk $\log 2$, while the refined split $s_{c(z)}=1$ versus
$s_{c(z)}=0$ has risk $H_{\mathrm{bin}}(\eta)=0.098$. Crucially, every
candidate bit $g_k(s)=s_k$ has the same marginal frequency, and only
$g_{c(z)}$ changes the conditional consequence distribution.

DGM-HQS keeps a repairable Bernoulli memory at each fine node and scores
candidate edges using disjoint proposal and scoring buffers of size 64, with 20
independent spurious Bernoulli candidates in the pool. An edge is accepted only
when its held-out log-loss gain exceeds the edge penalty and threshold.

This result is not a claim that DGM beats every possible learner; crossed
interaction features and a local stump also succeed. The point is structural
efficiency and consequence-sensitive refinement. DGM accepts one edge per fine
class and pools the two outcomes of the relevant bit, while crossed logistic
allocates one interaction parameter for every fine-query pair and exact caches
cannot compress the rule across random masks.

\subsection{Guarded decisions and graph-memory ablations}

The guarded refine-or-repair experiment tests whether refinement is
consequence-sensitive rather than novelty-sensitive. We create two old atoms. In
the repair atom, $Y\sim\mathrm{Bernoulli}(0.8)$ and all candidate distinctions
are independent of $Y$, but the current local prediction is deliberately stale
at $q(Y=1)=0.2$. The correct action is repair. In the refinement atom,
$Y\mid U=0\sim\mathrm{Bernoulli}(0.1)$ and
$Y\mid U=1\sim\mathrm{Bernoulli}(0.9)$, so the correct action is to refine by
$U$. The candidate pool contains the true distinction $U$, twenty independent
noise bits, and six geometry-only bits that are visually separable but
independent of $Y$. Candidate proposal and validation use separate buffers.

\subsection{Non-oracle CIFAR-100 Image-HQS}
\label{sec:image-hqs}

We next instantiate a harder CIFAR-100 hierarchy-query stream that removes the
oracle fine-label atom. Each round provides a frozen image embedding
$z=\phi(x)$ and an independently sampled mask $s\in\{0,1\}^{20}$. The binary
consequence is
\[
Y=s_{c(x)}
\]
with label flip probability $\eta=0.02$, where $c(x)$ is the CIFAR-100 coarse
class. The learner observes neither the fine label nor the coarse label before
or during its update; those labels are used only for diagnostics and oracle
ceilings. All methods share the same frozen prototype encoder. Online logistic
uses $[z,s]$, and crossed logistic uses $[z,s,z\otimes s]$.

In this non-oracle setting the image concepts learned online are not
coarse-pure, so a single local split bit is the wrong local action. We therefore
use image-only DGM routing with a mask-coordinate posterior memory at each
concept:
\[
P(Y=1\mid j,s)
\approx
\eta+(1-2\eta)\sum_{c=1}^{20}\pi_{j,c}s_c .
\]
After prediction, each selected concept $j$ receives soft credit $\alpha_j$ and
updates logits $\theta_j$, with $\pi_j=\mathrm{softmax}(\theta_j)$, by one
online gradient step on the Bernoulli NLL of
$\eta+(1-2\eta)\pi_j^\top s$. No fine label, coarse label, or true coordinate is
used in this update.

Paired bootstrap intervals support the NLL gaps. DGM improves test NLL over
online logistic by $0.0180$ nats with 95\% bootstrap CI $[0.0148,0.0212]$, and
over crossed logistic by $0.0085$ nats with CI $[0.0047,0.0122]$. Diagnostics
separate this result from the oracle-partition hierarchy recovery experiment:
the soft-route coarse-posterior diagnostic is $0.6555$, the $k$NN
coarse-posterior ceiling is $0.6522$, and the posterior hit rates for the true
mask coordinate are $0.2808/0.5440/0.6599$ at ranks $1/3/5$ versus random
$0.05/0.15/0.25$. Thus this result supports local posterior repair under
learned visual concepts, not exact recovery of the CIFAR hierarchy graph from
images. A small ablation in Appendix~\ref{app:image-hqs-audit} shows that
categorical and single-bit memories fail, global posterior memory is weaker
than local DGM posterior, and setting $\eta=0$ barely changes NLL.

\subsection{Trust aggregation and runtime baselines}

The calibration diagnostic isolates the calibration issue. We run a controlled
next-token stream with repeated contexts and a phase drift. The base predictor
is the phase-level unigram law, while the DGM memory routes by context and
stores token counts. An unsmoothed, overconfident memory improves accuracy but
can assign catastrophic probability to unseen post-drift tokens; a smoothed
memory is safer, and the trust aggregator mixes a grid
$\Lambda=\{0,0.05,0.1,0.2,0.4,0.7,1\}$ containing the frozen base predictor.
In the 4000-step run used as a smoke check, the base average NLL is $3.865$,
the unsmoothed overconfident memory average NLL is $5.173$, the smoothed
memory-only average NLL is $2.815$, and Hedge trust aggregation obtains
$2.763$. The logged trajectory satisfies the theorem's fallback bound with
numerical slack.

The relevant backpropagation baseline depends on the runtime contract. Full
offline backpropagation is a batch upper bound: it can revisit all examples and
change the representation quotient. The direct runtime baseline is online
backpropagation, which sees each example once and makes one gradient update
after the prequential prediction
\citep{rumelhart1986learning,zinkevich2003online,shalev2012online}.
Figure~\ref{fig:dgm-vs-backprop} reports both, so the comparison boundary is
explicit.

\begin{figure}[!htbp]
\centering
\includegraphics[width=\linewidth]{experiments/publication_figures/dgm_vs_backprop.pdf}
\caption{Runtime baselines against backpropagation. (A) On contextual XOR and
synthetic sequence-map streams, DGM memory has lower prequential NLL than
one-step online backpropagation. (B) DGM also has higher test accuracy; offline

\label{fig:dgm-vs-backprop}
\end{figure}

On the two-context XOR stream, online backpropagation obtains
$0.569\pm0.065$ prequential accuracy, $0.689\pm0.019$ prequential NLL, and
$0.654\pm0.038$ test accuracy. DGM obtains $0.804\pm0.025$ prequential
accuracy, $0.458\pm0.038$ prequential NLL, and $0.944\pm0.026$ test accuracy.
On the synthetic sequence-map stream, online backpropagation obtains
$0.113\pm0.029$ prequential accuracy, $3.065\pm0.061$ prequential NLL, and
$0.125\pm0.021$ test accuracy, while DGM obtains $0.862\pm0.005$,
$2.067\pm0.082$, and $0.912\pm0.077$, respectively. In the frozen-embedding
adapter check, ridge memory reaches $0.623\pm0.003$ MNIST accuracy in
$0.0018\,\mathrm{s}$ and $0.397\pm0.010$ CIFAR-10 accuracy in
$0.0004\,\mathrm{s}$, compared with $0.332\pm0.009$ and $0.359\pm0.009$ for a
200-step backpropagation head taking about $0.032\,\mathrm{s}$.

\paragraph{Scope.}
These experiments do not claim that DGM is a finished competitive
language-modeling system, that runtime memory dominates unconstrained offline
training, or that the non-oracle Image-HQS result establishes a full
loss-latency-memory frontier against nonparametric and gradient baselines. They
test the mechanisms required by the theory and provide predictive evidence
under a shared frozen encoder; stronger nonparametric baselines and full
systems accounting remain future work.
