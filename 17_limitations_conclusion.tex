\section{Limitations and Conclusion}
\label{sec:main-limitations}

\begin{itemize}
\item CGA's dense read is stable at small concept budgets, but large concept
banks will need fused kernels and eventually sparse concept retrieval.
\item The current retention decay is trainable and PoST-parameterized by
default, but its best time-scale allocation for large language modeling remains
empirical.
\item Fixed-bank CGA removes online slot-allocation complexity, but it makes
the concept count $C$ a real capacity hyperparameter.
\item CGA's test-time learning is prefix-local in the current formulation: the
concept value memory is reset per sequence unless a separate persistent-memory protocol
is added.
\item The 180M--440M language-model runs are the intended SOTA-readiness test,
but they still require completed training and latency reporting before the
paper can claim a large-scale win.
\end{itemize}

The empirical claim is deliberately scoped: CGA is evaluated as an
attention-like causal sequence mixer on MQAR and matched language-model
pretraining.
MQAR should show whether the concept bank can write and retrieve key-value
information under matched state budgets.
CGA-LM should show whether the same mechanism can replace token attention in a
trainable decoder at 180M--440M scale.
If CGA loses to optimized recurrent mixers, that is a real systems or modeling
gap rather than a wording problem.
The method still needs stronger fused kernels, retention-time-scale studies,
and distributed concept-bank implementations before it can be treated as a
fully optimized large-model systems result.

The main point is the memory-basis replacement.
Attention reads from token-indexed memory.
CGA reads from fixed concept identities whose value memories are updated by a
causal forward-time rule during inference.
This gives a clean attention-compatible path for testing whether concept-level
test-time learning can compete with token memory, recurrent state, and
TTT-style hidden-model updates under matched state budgets.
