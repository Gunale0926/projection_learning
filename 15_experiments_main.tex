\section{Experiments}
\label{sec:main-experiments}

The experiments are now organized around one question: can DGM serve as a
test-time learning layer, in the same broad comparison class as TTT-style
self-supervised adaptation and parameter-efficient adapters?
We therefore use one ML-facing protocol rather than a collection of unrelated
diagnostics.

\subsection{CIFAR-10 Test-Time Layer Benchmark}

We train a small CNN source model on CIFAR-10 and evaluate it on a stream of
test images under clean and corrupted conditions.
Each method receives the same pretrained source network and the same test
stream.
Prediction is measured before any update on the current batch.
The methods differ only in their runtime state and update rule.

\begin{itemize}
    \item \textbf{Source}: the frozen CNN with no runtime adaptation.
    \item \textbf{DGM}: the same CNN plus a graph-memory readout built from
    training embeddings.  The state is a sparse distinction graph with
    node-local class memories; prediction uses edge-gated retrieval.
    \item \textbf{DGM-Online}: the same graph-memory layer, additionally
    updated on the test stream using high-confidence pseudo-labels.  This is a
    forward-only test-time update: no backward pass is run at inference time.
    \item \textbf{TTT-Rotation}: a test-time training baseline with an auxiliary
    rotation-prediction task.  At test time it runs gradient steps on the
    self-supervised rotation loss before making the supervised prediction.
    \item \textbf{LoRA-Entropy}: a parameter-efficient test-time adapter
    baseline.  The source CNN is frozen and a low-rank residual adapter in the
    embedding space is updated by entropy minimization.
    \item \textbf{EntropyMin}: a standard online entropy-minimization
    adaptation baseline on the source model.
\end{itemize}

This comparison is intentionally narrower than a general CIFAR-10 benchmark.
It tests runtime learning mechanisms under a matched source architecture:
TTT changes a small model state by gradient descent, LoRA changes a low-rank
adapter by gradient descent, and DGM changes an explicit graph-memory state by
write and refinement operations.
The main reported metrics are accuracy, negative log likelihood, evaluation
time, memory state size, accepted concepts, and accepted distinction edges.

The source artifact for this protocol is
\path{experiments/run_cifar_ttt_dgm_benchmark.py}.
The intended publication run uses full CIFAR-10 training data, five paired
seeds, clean plus corruption streams, and the same batch order for every method.
Small smoke runs are used only to validate implementation; they are not paper
evidence.

\subsection{What Counts as a Positive Result}

The cleanest positive result is not that DGM beats every gradient-based
test-time method on every corruption.
TTT-style methods can be especially strong when the auxiliary task matches the
corruption.
The result we need is more specific:
\begin{enumerate}
    \item DGM improves over the frozen source model under the same backbone.
    \item DGM obtains competitive accuracy or log loss on at least one
    corruption regime while using no inference-time backward pass.
    \item DGM's runtime state is interpretable as a graph: accepted concepts,
    accepted edges, and memory bytes must be reported.
    \item When DGM loses to TTT or LoRA, the paper reports the loss honestly as
    a tradeoff between forward-only graph-memory updates and gradient-based
    test-time adaptation.
\end{enumerate}

This is the right SOTA-ready framing.
DGM is not presented as a replacement for all backpropagation.
It is presented as a runtime graph-memory layer with a different fast-state
contract from TTT and LoRA: explicit distinctions and local memories instead of
inner-loop gradient updates.
