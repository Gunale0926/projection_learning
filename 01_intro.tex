\section{Introduction}
\label{sec:intro}

Runtime learning is usually phrased as an update problem: after a model is surprised, which memory, cache entry, adapter, or parameter should be changed? This paper studies the prior structural question. Before deciding how to update, a learner must decide whether the observed error is representable under its current information.

At time $t$, a deployed learner has an information map
\begin{align*}
I_t:\Omega\to\calZ_t
\end{align*}
and a predictor
\begin{align*}
f_t=g_t\circ I_t .
\end{align*}
Fix a probability space $(\Omega,\calE,\mathbb P)$. The points of $\Omega$ are cases or histories; $\calE$ is the ambient sigma-algebra; and $\mathbb P$ is the data-generating distribution. The label space $\calZ_t$ is the learner's current quotient of the world. A label may be a decision-tree leaf, cache cell, hidden-state bucket, route, or runtime concept. The target is $Y:\Omega\to\calY$, predictions take values in an action space $\mathsf A$, and performance is measured by a loss $L:\calY\times\mathsf A\to\R$.

The information exposed by the current representation is
\begin{align*}
\calF_t=\sigma(I_t),
\end{align*}
the sigma-algebra of events decidable from $I_t(\omega)$. In a finite space this is the partition into fibers $I_t^{-1}(z)$. Hence the equality $I_t(\omega)=I_t(\omega')$ is not only a representational accident: every predictor preserving the current information must assign the same action to $\omega$ and $\omega'$.

We separate two update types. A \emph{repair} changes the downstream rule, local memory, or statistics while preserving information:
\begin{align*}
\sigma(I_{t+1})=\calF_t .
\end{align*}
A \emph{refinement} changes the information structure:
\begin{align*}
\sigma(I_{t+1})=\calG,\qquad \calG\supsetneq\calF_t .
\end{align*}
The point of the distinction is that the two operations have different formal power.

For a concrete predictor $f$, write
\begin{align*}
R_L(f)=\E[L(Y,f)].
\end{align*}
For an information structure $\calF\subseteq\calE$, let
\begin{align*}
\Pred(\calF)=\{a:\Omega\to\mathsf A: a\text{ is }\calF\text{-measurable}\}
\end{align*}
be the predictors admissible under $\calF$, and define the Bayes risk under information $\calF$ by
\begin{align*}
R_L(\calF)
=
\inf_{a\in\Pred(\calF)}\E[L(Y,a)].
\end{align*}
Given the current predictor $f_t$ and a candidate refinement $\calG\supseteq\calF_t$, the runtime excess risk decomposes into two adjacent gaps:
\begin{align}
\Delta_{\mathrm{repair}}(f_t,\calF_t)
&=R_L(f_t)-R_L(\calF_t),
\label{eq:intro-repair-gap}\\
\Delta_{\mathrm{refine}}(\calF_t,\calG)
&=R_L(\calF_t)-R_L(\calG).
\label{eq:intro-refine-gap}
\end{align}
Equivalently,
\begin{align}
R_L(f_t)-R_L(\calG)
&=
\underbrace{\Delta_{\mathrm{repair}}(f_t,\calF_t)}_{\text{best possible repair at fixed information}}
+
\underbrace{\Delta_{\mathrm{refine}}(\calF_t,\calG)}_{\text{value of new information}}.
\label{eq:intro-error-decomposition}
\end{align}
The repair gap is the error left by $f_t$ inside the current information structure. The refinement gap is the value of distinctions exposed by $\calG$ but absent from $\calF_t$.

The obstruction is simple and architecture-independent. Suppose $I_t(\omega)=I_t(\omega')$, but the two cases require different outputs. Every repaired predictor still factors through $I_t$, so it must assign the same output to both cases. No cache update, local optimizer, nearest-neighbor write, fast-weight repair, or test-time gradient step can remove this contradiction while preserving $\calF_t$. The missing object is not a better value inside the cell. It is the distinction that splits the cell.

The minimal corrective operation is canonical. If the learner accepts new evidence $U_t$, the least information update that makes that evidence usable is
\begin{align*}
\calF_{t+1}=\calF_t\vee\sigma(U_t).
\end{align*}
In a finite partition, this operation cuts exactly the old cells crossed by $U_t$ and leaves all other cells unchanged. Thus refinement is not arbitrary model growth. It is the least change in the sigma-algebra needed to make the missing evidence measurable. The value of a refinement is loss-dependent: under log loss it is conditional mutual information, and under squared loss it is conditional variance reduction. Because finite-sample refinement can overfit even when population information cannot hurt, an implementation should accept refinements only after a guarded gain test.

Distinction Graph Machines (DGM) are the algorithmic realization of this diagnostic. A DGM is a runtime data structure whose nodes are concepts with repairable memories and whose edges are binary distinction generators. Anchors provide stable concept identities, centroids support retrieval, and edges gate future routing. After observing a consequence, DGM scores empirical repair and refinement candidates. If the current concept is adequate, it repairs local memory at fixed information. If the consequence exposes an incompatibility inside the current information cell, it refines by adding a concept and consequence-forced distinction edges.

For language models, DGM-Embedding instantiates the same rule as a detached next-token memory layer. Hidden-state concepts store next-token embedding memories and produce sparse logit corrections through a differentiable read path. The write path is forward-only: after the next token is observed, the layer repairs selected memories or refines the concept graph without inference-time backpropagation.

This places DGM relative to several existing runtime mechanisms. Cache and $k$NN language models store or retrieve examples \citep{grave2017improving,khandelwal2020generalization}; fast weights and linear attention repair associative state \citep{ba2016using,schlag2021linear}; test-time adaptation optimizes an inference-time objective \citep{sun2020test,wang2021tent,sun2025learning}; online trees refine partitions \citep{breiman1984classification,domingos2000mining}; and adaptive prototype systems create categories \citep{carpenter1987massively,carpenter1987art2,kohonen1990self,fritzke1995growing,snell2017prototypical}. These ingredients are not new in isolation. The contribution is the formal refine-or-repair criterion: before writing runtime state, decide whether the observed error is representable under the current sigma-algebra or requires a new one.

\paragraph{Contributions.}
We make four contributions.
\begin{itemize}
    \item We formulate runtime learning through information maps, measurable predictors, and Bayes risk under a sigma-algebra of available distinctions.
    \item We decompose excess risk into a repair gap at fixed information and a refinement gap from missing information, and prove the missing-distinction obstruction.
    \item We characterize minimal refinement as $\calF_t\vee\sigma(U_t)$ and identify standard value formulas for log loss and squared loss.
    \item We introduce DGM and DGM-Embedding as guarded runtime algorithms that repair local memories or refine the information structure when consequences force a distinction.
\end{itemize}
