\section{Introduction}
\label{sec:intro}

A deployed model reads a long document. Early in the document, ``Jordan'' refers to a person; later, ``Jordan'' refers to a country. A cache can store both observations. A nearest-neighbor memory can retrieve similar contexts. A test-time optimizer can update a hidden state. But if the model treats both situations as the same runtime concept, every local update is forced to write incompatible consequences into one place. The failure is not that the memory was not updated enough. The failure is that the model never created the distinction.

Runtime learning therefore has two failure modes. In the first, the model is using the right concept but the concept's local state is stale. The right operation is repair: update the cache entry, sufficient statistic, ridge solution, fast-weight memory, or local adapter. In the second, the concept itself is too coarse: it merges states whose consequences differ. The right operation is refinement: create a new distinction before storing the new local state. In short,
\begin{align*}
\textit{if the atom is wrong, refine; if the local state is wrong, repair.}
\end{align*}

Modern runtime-learning systems usually implement one generic update response. Cache and $k$NN language models store or retrieve more examples \citep{grave2017improving,khandelwal2020generalization}. Fast weights and delta-rule linear attention repair associative state \citep{ba2016using,schlag2021linear}. Test-time training and adaptation optimize an inference-time surrogate \citep{sun2020test,wang2021tent,sun2025learning}. Online trees refine leaves by split scores \citep{breiman1984classification,domingos2000mining}. Sparse MoE routes to trained experts \citep{shazeer2017outrageously,fedus2022switch}. Adaptive category and prototype systems create or move categories \citep{carpenter1987massively,kohonen1990self,fritzke1995growing,snell2017prototypical}. These systems are useful, but they do not make the refine-or-repair decision explicit.

The distinction is structural. A learner's information state can be represented by a partition or sigma-algebra. Its admissible predictors are measurable with respect to that information. If two states lie in the same atom but require different outputs, no predictor over the current information can satisfy both. More computation inside the atom cannot solve the contradiction. This theorem is the paper's central diagnosis: a contradiction inside an atom is not a bad update; it is evidence that the atom should not exist.

Distinction Graph Machines implement this diagnosis as a runtime data structure. If contradictions require distinctions, the learner needs runtime concepts. If concepts must remain stable, each concept needs a frozen anchor. If retrieval must adapt, each concept also needs a movable centroid. If distinctions are local, the data structure should use graph edges rather than a single tree path. If concepts have local state, each node should carry a repairable memory. A DGM therefore maintains anchored concepts, movable retrieval centroids, local memories, and distinction edges. Edges are not metadata: they are binary generators that define observable distinctions and gate future routing. When a consequence arrives, DGM either repairs local memory or creates a new anchored concept and edges.

The formal core is minimal information refinement. In a finite space, adding a binary distinction cuts exactly the old atoms crossed by that distinction and leaves all other atoms unchanged. In a general measurable space, the corresponding update is the join $\calF_{t+1}=\calF_t\vee\sigma(Z_t)$. This is the smallest information update that makes new evidence observable, using standard language from measure-theoretic probability and statistical decision theory \citep{billingsley1995probability,kallenberg2002foundations,blackwell1953equivalent,berger1985statistical}. Refinement has exact value: under log loss it is conditional mutual information, and under squared loss it is conditional variance reduction. Population information cannot hurt, but finite-sample estimation can; DGM therefore uses statistically guarded refinement rather than splitting on every surprise.

For language models, the canonical layer is DGM-Embedding. Hidden-state concepts store next-token embedding memories and produce sparse logit corrections. The read path is differentiable, so slow model parameters can learn to use DGM memory. The write path is detached and forward-only: after observing the next token, the layer repairs selected local memories or refines the concept graph without inference-time backpropagation. Projection Memory and CPM appear as repair mechanisms once the distinction structure has been fixed.

\paragraph{Contributions.}
The paper makes four contributions.
\begin{itemize}
    \item \textbf{Diagnosis.} We show that some runtime errors are structurally unrepairable: if incompatible consequences lie inside one atom of the current information structure, no measurable predictor can satisfy them.
    \item \textbf{Principle.} We formalize minimal information refinement as the canonical operation that adds the missing distinction and nothing else, and quantify its value under log and squared losses.
    \item \textbf{Model.} We introduce DGM, a graph-based runtime learner whose edges generate distinctions and whose local memories support repair.
    \item \textbf{Layer.} We instantiate DGM as DGM-Embedding, a language-modeling layer with differentiable sparse reads and forward-only detached writes.
\end{itemize}
