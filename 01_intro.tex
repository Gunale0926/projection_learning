\section{Introduction}
\label{sec:intro}

Backpropagation gives a clean mathematical primitive for train-time learning. A differentiable model produces an error, and the chain rule distributes that error through the computation graph. This is the right language when learning means changing static parameters to reduce a global objective.

Runtime learning has a different demand. A deployed system already has state: memories, estimates, cached structure, and commitments induced by previous observations. New evidence should change the state only where the evidence forces change. If the update is too weak, the system fails to adapt; if it is too broad, the system destroys information that was not contradicted. The basic problem is therefore not merely descent of a loss, but controlled repair of state under evidence.

Projection Learning formalizes this repair principle. Let $S$ be a closed convex state space, let $\psi:S\to\R$ be differentiable and strictly convex, and define
\begin{align*}
D_\psi(u,v)=\psi(u)-\psi(v)-\ip{\nabla\psi(v)}{u-v}.
\end{align*}
If evidence at time $t$ gives a closed convex consistency set $\calC_t\subseteq S$, the hard update is
\begin{align*}
s_{t+1}=\argmin_{s\in\calC_t}D_\psi(s,s_t).
\end{align*}
If evidence is soft, represented by a convex loss $\ell_t:S\to\R$ and weight $\eta_t>0$, the update is
\begin{align*}
s_{t+1}=\argmin_{s\in S}\{D_\psi(s,s_t)+\eta_t\ell_t(s)\}.
\end{align*}
The guiding question is simple: among the states that satisfy, or best approximate, the new evidence, which one changes the old state the least?

The canonical neural instance studied here is Conservative Projection Memory. A runtime memory is a matrix $W_t\in\R^{m\times r}$, and a hidden state produces a key $k_t\in\R^r$ and value $v_t\in\R^m$. The evidence asks for $Wk_t=v_t$. If old keys form $K_{t-1}$, the safe write direction is the innovation
\begin{align*}
z_t=(I-\Pi_{t-1})k_t,\qquad \Pi_{t-1}=K_{t-1}K_{t-1}^\dagger,
\end{align*}
where $K_{t-1}^\dagger$ is the Moore-Penrose pseudoinverse. The soft conservative repair is
\begin{align*}
W_{t+1}=W_t+\frac{\eta_t}{1+\eta_t\norm{z_t}^2}(v_t-W_tk_t)z_t^\top.
\end{align*}
Thus the memory does not write along the raw key. It writes only along the part of the key not already explained by protected evidence.

This innovation rule is the core intuition of the paper. Large innovation means the key contains a new direction. Small innovation with small residual means the evidence is redundant. Small innovation with large residual means the evidence conflicts with what the memory has committed to preserve. The theory below turns this intuition into exact feasibility laws, optimality statements, and online algorithms.
