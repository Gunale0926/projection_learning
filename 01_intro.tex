\section{Introduction}
\label{sec:intro}

Backpropagation made hidden representations trainable by turning learning into differentiation through a static computation graph. This is the right primitive when the object being optimized is a parameterized function and the task is to distribute a scalar error through differentiable structure. Runtime learning asks for a different primitive. A deployed system receives evidence after it has already formed state, and the update must incorporate that evidence without changing what the evidence does not touch.

The central problem is therefore not only optimization but controlled repair. If the current state is $s_t$ and the evidence at time $t$ determines a set of acceptable states $\calC_t$, the natural update is
\begin{align*}
s_{t+1}=\argmin_{s\in\calC_t}D_\psi(s,s_t).
\end{align*}
If the evidence is noisy, the corresponding soft update is
\begin{align*}
s_{t+1}=\argmin_s\{D_\psi(s,s_t)+\eta_t\ell_t(s)\}.
\end{align*}
This paper studies these updates as a runtime learning principle. We call the principle Projection Learning: learning is minimal consistency repair.

The first contribution is a characterization theorem. Bregman projection is not merely one possible update. It is exactly the update that satisfies the evidence and obeys a Pythagorean inequality against every feasible ideal state. Thus the variational rule is equivalent to a stability statement: after projection, the learner has not moved away from any state that also satisfies the evidence.

The second contribution is a canonical neural realization called Conservative Projection Memory. A runtime memory is a matrix $W_t\in\R^{m\times r}$, and evidence is a key-value constraint $Wk_t=v_t$. If old keys form $K_{t-1}$, the correct write direction is not $k_t$ but the innovation
\begin{align*}
z_t=(I-\Pi_{t-1})k_t,\qquad \Pi_{t-1}=K_{t-1}K_{t-1}^+.
\end{align*}
The soft update is
\begin{align*}
W_{t+1}=W_t+\frac{\eta_t}{1+\eta_t\norm{z_t}^2}(v_t-W_tk_t)z_t^\top.
\end{align*}
This is the smallest update that repairs the new residual while preserving old constraints whenever such preservation is possible.

The innovation view gives the mechanism its semantics. If $\norm{z_t}$ is large, the key contains new information and the state can be updated in a new direction. If $z_t=0$ and the implied value agrees with the evidence, the observation is redundant. If $z_t=0$ and the value disagrees, the evidence is contradictory: no update can satisfy it while preserving the old bindings. The method therefore exposes novelty and conflict as algebraic facts, not as hidden consequences of an optimizer.
