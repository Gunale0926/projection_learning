\section{Introduction}
\label{sec:intro}

Runtime learning is usually phrased as an update problem: after a model is surprised, which memory, cache entry, adapter, or parameter should be changed? This paper studies the prior structural question. Before deciding how to update, a learner must decide whether the observed error is representable under its current information.

The central object of this paper is not a memory update, but the equivalence relation induced by the learner's runtime state. If two cases are assigned to the same runtime state, then any update that preserves that state space must treat them as indistinguishable. Repair modifies values on equivalence classes. Refinement modifies the equivalence relation itself.

Consider the smallest example. Let $\Omega=\{0,1\}$, let $P$ be uniform, and let the consequence be $Y=\omega$. If the current runtime representation has the trivial partition $\Pi=\{\{0,1\}\}$, then under log loss the best repair-only predictor assigns probability $1/2$ to each label and pays $\log 2$ per round. The refined partition $\Pi^\star=\{\{0\},\{1\}\}$ pays zero in the noiseless case. This gap is not caused by a bad optimizer, cache size, or local parameter value. It is caused by identifying two states that the target separates.

We formalize this as \emph{online prediction with refinable partitions}. A learner maintains a state
\begin{align*}
S_t=(\Pi_t,a_t),
\end{align*}
where $\Pi_t$ is a finite partition of the case space and $a_t:\Pi_t\to\mathsf A$ is a predictor on its atoms. On each round, the learner observes only the current atom containing the sampled case, predicts, observes the consequence, and then chooses either repair or refinement. A repair changes $a_t$ while keeping $\Pi_{t+1}=\Pi_t$. A refinement chooses an admissible binary distinction $U$ and replaces the partition by the join $\Pi_{t+1}=\Pi_t\vee U$.

This model makes a lower bound immediate and useful. For any fixed partition $\Pi$, every repair-only learner whose predictions remain $\Pi$-measurable has expected per-round loss at least the Bayes risk $R_L(\Pi)$. Relative to an oracle using a refined partition $\Pi^\star$, its expected regret is at least
\begin{align*}
T\left(R_L(\Pi)-R_L(\Pi^\star)\right).
\end{align*}
Thus, whenever the refinement gap is positive, repair-only learning has linear regret against a refined oracle. Caches, local memories, fast weights, nearest-neighbor writes, and test-time optimizers fall under this statement only when their updates preserve the relevant partition. Methods that change the representation are refinements in this model, not repairs.

The minimal escape from this lower bound is also canonical. Adding a binary distinction $U$ replaces the current partition by
\begin{align*}
\Pi\vee U
=
\{B\cap U^{-1}(0),B\cap U^{-1}(1):B\in\Pi\}\setminus\{\varnothing\}.
\end{align*}
This operation cuts exactly the atoms crossed by $U$ and leaves all other atoms unchanged. A distinction is therefore not merely an auxiliary feature; it changes the information structure on which future predictions are allowed to depend.

Not every new bit is worth adding. A refinement should be accepted only if it changes the best achievable decision. We characterize this condition in Bayes-risk terms: if $\calF\subseteq\calG$ and the $\calG$-Bayes action is unique, then $R_L(\calF)-R_L(\calG)=0$ iff that refined Bayes action already has an $\calF$-measurable version. Under log loss, the gap becomes an expected conditional KL divergence, and a binary distinction has value $I(Y;U\mid\calF)$. This gives a consequence-based split score: accept distinctions because they change outcome distributions, not because they are geometrically novel.

Distinction Graph Machines (DGM) are the data-structure realization of this abstract model. The formal object is Hard-DGM: fixed anchors and accepted binary edges generate an edge skeleton, anchor-distance ordering completes the exact hard route, node memories are locations selected by that hard-route map rather than atoms in one-to-one correspondence with the edge partition, and refinement adds a new anchored concept together with its edge and order information. The practical implementation is Soft-DGM: centroids, sparse retrieval, and differentiable edge gates approximate the hard route. The partition-level guarantees apply exactly to Hard-DGM and to Soft-DGM only as an approximation under candidate-recall and margin conditions.

\paragraph{Contributions.}
We make six contributions.
\begin{itemize}
    \item \textbf{Model.} We introduce online prediction with refinable information partitions, where repair preserves $\Pi_t$ and refinement replaces $\Pi_t$ by $\Pi_t\vee U$.
    \item \textbf{Lower bound.} We prove that any repair-only learner suffers linear regret against a refined oracle whenever $R_L(\Pi)>R_L(\Pi^\star)$.
    \item \textbf{Characterization.} We show that a refinement has positive value exactly when it changes the Bayes action; under log loss, the value of a binary distinction is $I(Y;U\mid\calF)$.
    \item \textbf{Algorithmic guarantees.} We give bounded-loss statistically guarded split tests, distinction-complexity parameters, realizable greedy recovery guarantees, and a finite-class oracle inequality.
    \item \textbf{Data structure.} We implement refinable information as DGM, where graph edges are binary generators, anchor order completes the exact hard route, and node memories implement repair.
    \item \textbf{Separating examples.} We give controlled streams showing that repair-only learners fail on missing-distinction tasks while refine-and-repair learners recover the needed partition. These are mechanism checks, not claims of language-model-scale competitiveness.
\end{itemize}
