\section{Introduction}
\label{sec:intro}

Runtime learning has two different failure modes. Sometimes the model is looking at the right kind of state, but the state is numerically wrong; then the right operation is repair. Sometimes the model is using an information structure that identifies situations whose consequences differ; then no local repair can solve the error. The right operation is refinement.

This paper builds a model around that distinction. A Distinction Graph Machine keeps a growing set of prototypes, local memories, and distinction edges. The prototypes define what the learner currently treats as concepts. The edges are binary generators: together they induce a partition of hidden-state space and gate which local memories are compatible with a query. The local memories store what the learner currently believes inside each concept. On an input $x$, an encoder produces $h=\phi_\theta(x)$, the machine retrieves nearby prototypes, edge-gates their routing weights, and their local memories predict. When the consequence $y$ is observed, the machine decides whether to create a new prototype and distinction edges, or repair the memory of an existing prototype.

The guiding rule is
\begin{align*}
\text{if the atom is wrong, refine; if the local state is wrong, repair.}
\end{align*}
Refinement creates a new concept when existing concepts are too coarse. Repair updates a local model inside the existing graph. For a simple classifier, repair can be an exemplar or sufficient-statistic update. For a richer local memory, repair can be Projection Memory or CPM.

The theory explains why the two operations should not be collapsed. If two states lie in the same atom but require different outputs, no measurable predictor over the current partition can satisfy both. This is not an optimization failure; it is a missing distinction. Conversely, if the current concept is adequate and only its local state is stale, creating a new concept is unnecessary. The model should update the local memory.

We formalize both sides. Minimal information refinement is the smallest partition or sigma-algebra update that makes new evidence observable. Its value is exact under standard losses: conditional mutual information for log loss and conditional variance reduction for squared loss. The DGM data structure adds the engineering layer missing from the pure refinement principle: graph-induced partitions, edge-gated routing, canonical local memories, penalized refine-or-repair rules, language-modeling integration, and train/test APIs. We also describe DGM Transformer layers whose read paths are differentiable and whose write paths are detached, so that large models can learn to use DGM memory during training without backpropagating through the entire memory history.
