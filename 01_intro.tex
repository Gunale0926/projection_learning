\section{Introduction}
\label{sec:intro}

Runtime learning is often formulated as an update problem: after a surprising observation, which part of the learner's state should change? Existing mechanisms include cache and retrieval memories \citep{grave2017improving,khandelwal2020generalization,lewis2020retrieval}, parameter-efficient adaptation states \citep{houlsby2019parameter,hu2022lora}, and test-time optimization \citep{sun2020test,wang2021tent}. This paper studies a prior question. Before choosing an update rule, a learner must determine whether the error is expressible under its current information structure. If two cases are indistinguishable to the runtime state but require different predictions, no repair of values inside that state can separate them.

We model this distinction using information partitions, following the decision-theoretic view that available information constrains admissible decision rules \citep{blackwell1953equivalent,berger1985statistical}. A runtime state induces an equivalence relation over cases. A repair update changes the action assigned to an existing equivalence class. A refinement update changes the equivalence relation itself by adding an observable distinction. In the simplest log-loss example, the trivial partition on two noiseless labels pays $\log 2$ per round, while the partition that separates the two states pays zero. The gap is not an optimization failure; it is an information-structure error.

The first result is a repair obstruction for online prediction. For any fixed partition $\Pi$, every adaptive learner whose predictions remain $\Pi$-measurable has expected loss at least the Bayes risk $R_L(\Pi)$ on each round. Relative to an oracle that predicts using a refined partition $\Pi^\star$, the cumulative regret is therefore at least
\begin{align*}
T\left(R_L(\Pi)-R_L(\Pi^\star)\right),
\end{align*}
up to the standard epsilon-oracle relaxation when Bayes actions are not attained. Thus a positive refinement gap is a structural lower bound: updates that preserve $\Pi$ cannot remove it. Gradient steps, cache writes, local sufficient-statistic updates, and adapter updates fall under this statement only when they preserve the relevant information map; updates that change the representation are refinements in this model, not repairs.

The second question is which refinements are worth accepting. The minimal way to add a binary distinction $U$ is the join
\begin{align*}
\Pi\vee U
=
\{B\cap U^{-1}(0),B\cap U^{-1}(1):B\in\Pi\}\setminus\{\varnothing\},
\end{align*}
or, in sigma-algebra form, $\calF\vee\sigma(U)$ \citep{billingsley1995probability,kallenberg2002foundations}. This operation splits exactly the atoms crossed by the new distinction and leaves the rest unchanged. We then characterize useful refinements in Bayes-risk terms. If $\calF\subseteq\calG$ and the $\calG$-Bayes action is unique, the refinement gap is zero exactly when that refined Bayes action already has an $\calF$-measurable version. Under log loss, the value of a binary distinction is $I(Y;U\mid\calF)$; under squared loss, it is conditional variance reduction \citep{cover2006elements,gneiting2007strictly}. Refinement is therefore consequence-sensitive rather than purely geometric.

We instantiate this escape operation as Distinction Graph Machines (DGM). DGM is a runtime graph-memory layer attached to an encoder such as a transformer, CNN, or MLP \citep{vaswani2017attention,lecun1998gradient,rumelhart1986learning}. The encoder supplies hidden states $h=\phi_\theta(x)$; DGM maintains graph nodes with repairable local memories and graph edges corresponding to accepted binary distinctions. The formal Hard-DGM state gives an exact graph-generated information structure, while Soft-DGM provides a sparse differentiable approximation under candidate-recall and margin assumptions. This scope is deliberately limited: DGM is not claimed to dominate unrestricted end-to-end neural adaptation. It is an algorithmic realization of a refine-or-repair decision when the runtime learner has an explicit information structure.

\paragraph{Contributions.}
We make five contributions.
\begin{itemize}
    \item \textbf{Model and obstruction.} We introduce online prediction with refinable information partitions and prove a linear repair-only regret lower bound against refined oracles.
    \item \textbf{Refinement principle.} We identify minimal partition and sigma-algebra joins as the canonical operation for adding a distinction.
    \item \textbf{Value characterization.} We show that useful refinements are exactly those that change the Bayes action, with conditional mutual information and conditional variance reduction as the log-loss and squared-loss special cases.
    \item \textbf{Algorithms and guarantees.} We give guarded empirical split rules, distinction-complexity parameters, realizable greedy recovery guarantees, and a finite-class oracle inequality.
    \item \textbf{Data structure and checks.} We implement the theory as DGM, a graph-memory data structure with repairable node memories and distinction edges, and give both an oracle-partition CIFAR-100 hierarchy query stream and a non-oracle image-HQS stream with diagnostics that separate repair-only learners from refine-and-repair learners.
\end{itemize}
