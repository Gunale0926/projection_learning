\section{CALM: Conceptual Attention Language Modeling}
\label{sec:large-model-training}

For language modeling, the world state is a prefix and the consequence is the next token. The Conceptual Attention Language Model (CALM) replaces the causal attention sublayer inside a decoder block with CA:
\begin{align*}
\operatorname{CABlock}(X)
=
X+\operatorname{CA}(\operatorname{Norm}(X)).
\end{align*}
The block has the same external shape contract as attention, $[B,L,d_{\rm model}]\mapsto[B,L,d_{\rm model}]$. The residual stream, layer normalization, embeddings, and language-model head are otherwise standard decoder components. In a Transformer-style decoder, CA can be paired with the usual MLP sublayer. For the state-matched PoST LM runs, the default implementation uses the same mixer-only residual scaffold as the Mamba-2 baseline; an MLP sublayer is an explicit configuration option rather than part of the main comparison.

\subsection{Fixed concept memory}

At layer $\ell$, CALM keeps a fixed bank of $C$ concept identities $a_{\ell hj}\in\mathbb{R}^{d_a}$ for each head $h$ and concept slot $j$. These identities are fixed addresses shared across sequences. The online state is the value memory
\begin{align*}
M_t^\ell\in\mathbb{R}^{H\times C\times d_v^c}.
\end{align*}
Each token produces read queries, write keys, write values, write gates, and retention-decay projections:
\begin{align*}
q_t=Q_\ell x_t,\qquad k_t=K_\ell x_t,\qquad v_t=V_\ell x_t,\qquad
g_t=\sigma(w_g^\top x_t+b_g).
\end{align*}
The read is causal: token $t$ reads only concept values produced by earlier writes. For concept slots $j=1,\ldots,C$,
\begin{align*}
\alpha_{thj}
=
\operatorname{softmax}_{j}
\left(\gamma_\ell q_{th}^\top a_{\ell hj}\right),
\qquad
z_{th}=\sum_{j=1}^{C}\alpha_{thj}M_{t,hj}^\ell,
\qquad
y_t=O_\ell[z_{t1};\ldots;z_{tH}].
\end{align*}
Multi-head CA uses independent address and value subspaces per head and concatenates the head outputs before the output projection, exactly as multi-head attention does.

\subsection{Read before write}

The update order is the same during training and autoregressive inference:
\begin{align*}
\text{read from }M_t^\ell
\rightarrow
\text{emit }y_t
\rightarrow
\text{decay and write }(k_t,v_t,g_t)
\rightarrow
\text{update }M_{t+1}^\ell .
\end{align*}
This prevents a token from reading its own write. The inference-time update is the test-time learning mechanism in CALM: the model weights are fixed, but the concept value memory changes after the causal write and future tokens condition on that updated memory.

For a delayed write key $\bar k_t=k_{t-\delta}$, CA computes write weights
\begin{align*}
\beta_{thj}
=
\operatorname{softmax}_{j}
\left(\gamma_\ell \bar k_{th}^\top a_{\ell hj}\right).
\end{align*}
With retention factor $\rho_{th}$, the token-level streaming update is
\begin{align*}
M_{t+1,hj}^{\ell}
=
\rho_{th}M_{t,hj}^{\ell}
+g_t\beta_{thj}v_{th}.
\end{align*}
For parallel training, CA groups tokens into causal blocks. All tokens in a block read the memory from before the block; after the reads, the block writes are accumulated into one affine update. For block $b=[s,e)$, define
\begin{align*}
U_{b,hj}^{\ell}
=
\sum_{t=s}^{e-1}
\mathbf{1}_{t\ge \delta}
g_t\beta_{thj}v_{th},
\qquad
\rho_{bh}^{\ell}
=
\prod_{t=s}^{e-1}\rho_{th}^{\ell}.
\end{align*}
The block transition is
\begin{align*}
M_{b+1,hj}^{\ell}
=
\rho_{bh}^{\ell}M_{b,hj}^{\ell}
+U_{b,hj}^{\ell}.
\end{align*}
Thus each block is summarized by an affine map $(\rho_b,U_b)$. These summaries compose associatively:
\begin{align*}
(\rho_2,U_2)\odot(\rho_1,U_1)
=
(\rho_2\rho_1,\ U_2+\rho_2U_1),
\end{align*}
with identity $(1,0)$. The prefix memory before every block can therefore be recovered by an exclusive associative scan over the block summaries. This is exact for the block-causal CA layer and exposes $O(\log N_{\rm block})$ cross-block parallel depth. Setting block size one recovers the exact token-streaming recurrence; using block size $B>1$ gives a coarser causal schedule in which writes become visible to later blocks rather than to later tokens inside the same block.

Both MQAR and language-model variants use a learned write gate initialized with negative bias and regularized by a small L1 penalty. The current experiments also use a trainable retention factor with a PoST-style ordered decay parameterization, input-dependent step sizes in a bounded range, and optional position-adaptive scaling. These are parameterization choices for stable allocation of forgetting time scales, not the definition of CA itself; CA can be run with a free retention parameterization for ablations.

\subsection{Training boundary}

Slow neural parameters are trained by ordinary next-token backpropagation through both the CA read and the continuous write. In the fixed-bank implementation used by CALM, gradients flow through the read weights, write weights, value projection, write gate, retention factors, output projection, and the block-to-block value-memory recurrence. The dynamic graph bookkeeping path can detach routing state, but that is not the main fixed-bank experimental path. At inference time there is no gradient step and no update to model weights. The only state that changes is the forward-updated concept value memory. Thus CALM is not a backprop-free language model and not a test-time optimizer over hidden model weights. It is an attention replacement whose fast state is a bounded concept memory.

\begin{algorithm}[H]
\caption{\textsc{CALayerForward}}
\label{alg:calm}
\begin{algorithmic}[1]
\State \textbf{Input:} hidden states $x_{1:L}$, fixed concept identities
$a_{1:C}$, zero value memory $M_0$
\State project all queries, write keys, values, gates, and retention rates
\State form each block summary $(\rho_b,U_b)$ independently
\State compute all block prefix memories by exclusive associative scan over summaries
\State read the scanned prefix memory inside each block with continuous concept attention
\State emit block outputs through the output projection
\State \Return contextualized hidden states
\end{algorithmic}
\end{algorithm}

\subsection{Scaling path}

The mathematical scaling path is the affine block scan above. The reported experiments use dense concept reads and dense block writes, which is appropriate for state-equalized MQAR and for the first language-model runs with $C=128$. An optimized system can fuse concept scoring, softmax, block-summary construction, the associative scan, and block reads, but this is an implementation of the same equations rather than a separate algorithm. For substantially larger concept banks, sparse concept retrieval and sharded concept banks become systems work. The architectural claim is independent of those choices: CA changes the memory basis of attention from tokens to concepts, while the scan algebra supplies the parallel recurrent evaluation.

\subsection{Evaluation protocol}

We use the Language Model Evaluation Harness~\citep{gao2024framework} in a zero-shot setting. The reported tasks are LAMBADA, HellaSwag, PIQA, ARC-Easy, ARC-Challenge, WinoGrande, and OpenBookQA, matching the protocol in Section~\ref{sec:main-experiments}. LAMBADA perplexity is not mixed into the main zero-shot table; final next-token perplexity is measured by \path{experiments/eval_final_ppl.py} on the last $n$ samples of the same corpus used for training, using the training tokenizer and non-overlapping $2048$-token windows. For reproducibility and systems comparison, each checkpoint should report: final same-corpus negative log likelihood and perplexity, zero-shot task metrics, training tokens per second, prefill latency, decode latency, parameter count, online state size, and CA diagnostics for write gate, read scale, retention, and decay. The latency table is generated by \path{scripts/benchmark_lm_latency.py}, which uses synthetic tokens so that all checkpoints are measured with identical batch size, context length, dtype, and hardware.
