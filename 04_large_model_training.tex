\section{DGM Layers for Large-Model Training}
\label{sec:large-model-training}

The data structure above is useful as a runtime learner, but a large model needs a layer. The layer must satisfy two constraints that pull in opposite directions. Its read computation must be differentiable, so that the backbone can learn to use the memory. Its write computation must be cheap and stable, so that training and inference can use the same update rule without backpropagating through the entire memory history.

\subsection{Common layer interface}

Let $h_t\in\R^d$ be the hidden state at a Transformer layer and let
\begin{align*}
q_t=Qh_t\in\R^{d_q}
\end{align*}
be the DGM query. A DGM layer maintains query-space prototypes $c_j\in\R^{d_q}$, local memories, counts, and distinction edges. Retrieval gives $N_k(q_t)$ and the read path uses the edge-gated weights from \eqref{eq:edge-gated-routing}, with $q_t$ in place of $h$. The read path is differentiable through the selected memories, routing weights, and backbone parameters. The top-$k$ retrieval itself may be treated as a sparse selection operator, as in mixture-of-experts and retrieval systems.

\subsection{DGM-Embedding layer}

The simplest large-model layer stores an output embedding mean at each prototype:
\begin{align*}
\calS_t^{\mathrm{emb}}=\{c_j,r_j,n_j\}_{j=1}^{N_t}\cup G_t,
\qquad
r_j\in\R^{d_e}.
\end{align*}
For language modeling, with output embedding $E[y]$, the DGM read contributes a logit correction
\begin{align*}
\Delta\ell_t(y)=\sum_{j\in N_k(q_t)}\alpha_j(q_t)E[y]^\top r_j.
\end{align*}
The model adds this to the backbone logits:
\begin{align*}
\ell(y\mid x_{\le t})=\ell_\theta(y\mid x_{\le t})+\lambda_t\Delta\ell_t(y).
\end{align*}
After the next token is observed, $v_t=E[x_{t+1}]$ gives the detached sufficient-statistic write
\begin{align*}
r_j^+=\frac{n_jr_j+\alpha_j(q_t)v_t}{n_j+\alpha_j(q_t)},
\qquad
n_j^+=n_j+\alpha_j(q_t).
\end{align*}
If the penalized refinement test in \eqref{eq:refine-accept} accepts a new concept, the layer creates a new prototype at $q_t$, initializes $r_{\mathrm{new}}=v_t$, and adds distinction edges to the confused prototypes.

\subsection{DGM-Adapter layer}

A richer layer stores a local linear repair map:
\begin{align*}
\calS_t^{\mathrm{adp}}=\{c_j,W_j,n_j\}_{j=1}^{N_t}\cup G_t,
\qquad
W_j\in\R^{d\times r}.
\end{align*}
Let $K:\R^d\to\R^r$ be a learned or fixed key projection and set $k_t=Kh_t$. The adapter read is
\begin{align*}
\Delta h_t=\sum_{j\in N_k(q_t)}\alpha_j(q_t)W_jk_t,
\qquad
\tilde h_t=h_t+\gamma_\theta(h_t)\Delta h_t.
\end{align*}
Given a local target vector $v_t\in\R^d$, the projected rank-one repair for a selected prototype is
\begin{align*}
W_j^+=W_j+
\frac{\eta\alpha_j(q_t)}
{1+\eta\alpha_j(q_t)\norm{k_t}_2^2}
\big(v_t-W_jk_t\big)k_t^\top.
\end{align*}
Equivalently, a memory may maintain small ridge sufficient statistics and solve the local adapter in closed form. If the implementation uses a low-rank factorization $W_j=B_jA_j$ for the read path, the closed-form write is a repair of $W_j$ or its sufficient statistics; it is not a simultaneous closed-form update of arbitrary factors $A_j$ and $B_j$.

\subsection{Detached write path}

The update is detached:
\begin{align*}
\operatorname{stopgrad}(\calS_{t+1})=\operatorname{DGMObserve}(\operatorname{stopgrad}(\calS_t),h_t,x_{t+1}).
\end{align*}
Thus the model trains slow parameters through the read path and learns fast state through the same observation rule used at inference. The layer is not a replacement for backpropagation through the backbone. It is a replacement for inference-time and history-time backpropagation through the adaptive state.

\begin{algorithm}[H]
\caption{\textsc{TrainStep} for a DGM Transformer layer}
\label{alg:dgm-train-step}
\begin{algorithmic}[1]
\State \textbf{Input:} token block $x_{1:L}$, slow parameters $\Theta$, DGM state $\calS$
\For{$t=1,\ldots,L$}
\State compute hidden state $h_t$ using the backbone
\State $q_t\gets Qh_t$
\State retrieve $N_k(q_t)$ from the prototype bank
\State compute edge-gated weights $\alpha_j(q_t)$
\State apply either the DGM-Embedding logit read or the DGM-Adapter hidden-state read
\EndFor
\State compute next-token loss $\calL(\Theta,\calS)$
\State update slow parameters $\Theta$ by backpropagation through the selected read computations
\For{$t=1,\ldots,L$}
\State update $\calS$ by detached \textsc{DGMObserve} using consequence $x_{t+1}$
\EndFor
\State \Return updated $\Theta$ and $\calS$
\end{algorithmic}
\end{algorithm}

\subsection{Why this can scale}

During training, backpropagation sees the selected read computation. During inference, the same layer performs forward retrieval and detached state repair. Therefore the train and inference programs differ only in whether slow parameters receive gradients.

\begin{theorem}[DGM layer cost, informal version of Theorem~\ref{thm:dgm-layer-cost:formal}]
\label{thm:dgm-layer-cost:informal}
Consider a DGM-Adapter layer with hidden dimension $d$, query dimension $d_q$, local rank $r$, top-$k$ retrieval, selected incident edge-gate count $Q_k$, and prototype-bank query cost $C_{\mathrm{ret}}(N_t,d_q)$. For one token, the read overhead is
\begin{align*}
O(dd_q+C_{\mathrm{ret}}(N_t,d_q)+Q_k+kdr).
\end{align*}
If the write path is detached and uses rank-one local repair, the write overhead is
\begin{align*}
O(dd_q+C_{\mathrm{ret}}(N_t,d_q)+Q_k+k(d_q+dr)).
\end{align*}
For a block of length $L$, the DGM state history need not be stored for backpropagation; the additional activation memory is the selected read computation, not the full sequence of memory states. The DGM-Embedding layer replaces the $kdr$ adapter term by the selected embedding-logit cost.
\end{theorem}

The important comparison is with test-time gradient adaptation. A TTT-style layer typically performs an inner backward pass at inference time. A DGM layer performs retrieval, sparse local read, and closed-form or sufficient-statistic repair. The cost is forward-only:
\begin{align*}
\text{DGM inference}=\text{backbone forward}+\text{retrieval}+\text{sparse local repair}.
\end{align*}

\subsection{Distributed training}

The prototype bank can be sharded by the same principle used in retrieval and sparse expert systems. Each device owns a subset of prototypes. A query is routed to a small set of shards, each shard returns local top-$k$ candidates, and a final top-$k$ merge selects the memories used by the read path. Only selected memories receive gradients. Detached writes are sent back to the owning shards.

Budgeting is necessary for long training. A practical DGM layer maintains
\begin{align*}
N_t\le N_{\max}.
\end{align*}
When the bank is full, the layer may merge compatible prototypes, evict low-usage prototypes, or compress old prototypes into a shared local memory. These operations are modeling choices rather than new learning principles. The principle remains refine when the representation is too coarse, and repair when the local state is adequate.

\subsection{Closed-form adapters}

DGM-Ridge is the main closed-form adapter used in our checks. Let $Z\in\R^{n\times d}$ be normalized support embeddings stored by a DGM memory and let $Y\in\R^{n\times C}$ be one-hot consequences. The adapter computes
\begin{align*}
\Gamma=(ZZ^\top+\lambda I)^{-1}Y,
\qquad
\operatorname{logits}(z)=zZ^\top\Gamma.
\end{align*}
This is a projection-style repair: it fits the local adapter in one linear solve rather than by iterated backpropagation. When $n$ is small, as in few-shot adaptation or a retrieved local neighborhood, the solve is cheap. In a large model, the same idea can be applied per prototype, per document, or per retrieved shard.
