\section{DGM-Embedding: A Runtime Memory Layer for Language Models}
\label{sec:large-model-training}

For language modeling, the world state is a prefix and the consequence is the next observed token. This makes the refine-or-repair decision concrete. If a prefix type is already represented, the layer should repair its next-token memory. If the same runtime concept is being asked to predict incompatible next-token consequences, the layer should refine the concept graph. DGM-Embedding is the canonical large-model layer because its detached target is directly observed: the next-token embedding.

\subsection{Hidden-state concepts}

Let a frozen or slowly trained backbone produce hidden states
\begin{align*}
h_t=\phi_\theta(x_{\le t}),
\end{align*}
and let
\begin{align*}
q_t=Qh_t\in\R^{d_q}
\end{align*}
be the DGM query. A DGM layer maintains frozen query-space anchors $a_j\in\R^{d_q}$, movable query-space centroids $c_j\in\R^{d_q}$, embedding memories $r_j\in\R^{d_e}$, counts, and distinction edges:
\begin{align*}
\calS_t^{\mathrm{emb}}=\{a_j,c_j,r_j,n_j\}_{j=1}^{N_t}\cup G_t.
\end{align*}
The formal information structure is generated by anchors and edges. Retrieval uses centroids as adaptive statistics, while edges are computed from anchors. The recall-oriented candidate set $N(q_t)$ is the same one used by DGM: centroid neighbors for adaptive locality, anchor neighbors for formal route recall, and graph neighbors for recent conflicts.

\subsection{Logit correction}

The read path is differentiable through the selected memories, routing weights, and backbone parameters. With output embedding $E[y]$, the DGM read contributes a sparse logit correction
\begin{align*}
\Delta\ell_t(y)=\sum_{j\in N(q_t)}\alpha_j(q_t)E[y]^\top r_j,
\end{align*}
where $\alpha_j$ is edge-gated as in \eqref{eq:edge-gated-routing}. Equivalently, define the mixed memory vector
\begin{align*}
r_{\mathrm{mix}}(q_t)=\sum_{j\in N(q_t)}\alpha_j(q_t)r_j.
\end{align*}
Then $\Delta\ell_t(y)=E[y]^\top r_{\mathrm{mix}}(q_t)$, and the final logits are
\begin{align*}
\ell(y\mid x_{\le t})=\ell_\theta(y\mid x_{\le t})+\lambda_t\Delta\ell_t(y).
\end{align*}
Here $\lambda_t$ is a learned or fixed trust weight. With tied output embeddings and $d_e=d$, the same correction can be implemented as
\begin{align*}
h_t^{\mathrm{DGM}}=h_t+\lambda_t r_{\mathrm{mix}}(q_t),
\end{align*}
followed by the usual LM-head multiplication. This avoids an extra vocabulary-sized projection.

\subsection{Read before observe}

DGM-Embedding uses the same streaming program during training and inference:
\begin{align*}
\text{predict }x_{t+1}\rightarrow\text{observe }x_{t+1}\rightarrow\text{repair or refine}\rightarrow\text{predict }x_{t+2}.
\end{align*}
After the next token is observed, $v_t=E[x_{t+1}]$ gives the detached sufficient-statistic write for every selected $j\in N(q_t)$, with all unselected memories unchanged:
\begin{align*}
r_j^+=\frac{n_jr_j+\alpha_j(q_t)v_t}{n_j+\alpha_j(q_t)},
\qquad
n_j^+=n_j+\alpha_j(q_t).
\end{align*}
Hard-update mode is the special case $\alpha_{j^\star}=1$ and $\alpha_j=0$ for $j\ne j^\star$. If the penalized refinement test in \eqref{eq:refine-accept} accepts a new concept, the layer creates a new anchor and centroid at $q_t$, initializes $r_{\mathrm{new}}=v_t$, and adds distinction edges to $\operatorname{Confused}_q(q_t,x_{t+1})$. Ordinary repairs may update centroids and memories but leave anchors fixed.

During streaming prediction, document continuation, user correction, tool feedback, or supervised next-token evaluation, the next observed token is genuine evidence and can refine or repair the runtime memory. During unconstrained generation, the model should not update from its own sampled token by default, because self-generated samples are actions, not external consequences.

\begin{algorithm}[H]
\caption{\textsc{ObserveNextToken} for DGM-Embedding}
\label{alg:dgm-lm}
\begin{algorithmic}[1]
\State \textbf{Input:} prefix $x_{\le t}$, observed token $x_{t+1}$, DGM state $\calS_t$
\State $h_t\gets\phi_\theta(x_{\le t})$ and $q_t\gets Qh_t$
\State retrieve $N(q_t)$ and compute edge-gated logits $\ell_\theta+\lambda_t\Delta\ell_t$
\State $j^\star\gets$ highest-weight prototype
\State $\rho\gets-\log p(x_{t+1}\mid x_{\le t})$
\If{$\rho\le\tau_{\mathrm{repair}}$}
\State update DGM-Embedding memories for all $j\in N(q_t)$ using soft credit $\alpha_j(q_t)$
\Else
\State accept a new anchored concept only if the penalized refinement gain test holds
\EndIf
\State \textbf{Hard variant:} set $\alpha_{j^\star}=1$ and $\alpha_j=0$ for $j\ne j^\star$
\State \Return updated state
\end{algorithmic}
\end{algorithm}

\subsection{Sequential detached training}

The update is detached:
\begin{align*}
\operatorname{stopgrad}(\calS_{t+1})=\operatorname{DGMObserve}(\operatorname{stopgrad}(\calS_t),h_t,x_{t+1}).
\end{align*}
In sequential detached training, the model trains slow parameters through the read path and learns fast state through the same observation rule used at inference. The layer is not a replacement for backpropagation through the backbone. It is a replacement for inference-time and history-time backpropagation through the adaptive state.

\begin{algorithm}[H]
\caption{\textsc{SequentialDetachedTrainStep} for a DGM-Embedding Transformer layer}
\label{alg:dgm-train-step}
\begin{algorithmic}[1]
\State \textbf{Input:} token block $x_{1:L}$, slow parameters $\Theta$, DGM state $\calS$
\State $\calL\gets0$
\For{$t=1,\ldots,L$}
\State compute hidden state $h_t$ using the backbone
\State $q_t\gets Qh_t$
\State retrieve $N(q_t)$ from the prototype bank
\State compute edge-gated weights $\alpha_j(q_t)$
\State apply the DGM-Embedding read
\State produce logits for $x_{t+1}$ and accumulate $\calL\gets\calL+\ell_t$
\State $\calS\gets\operatorname{stopgrad}(\textsc{DGMObserve}(\operatorname{stopgrad}(\calS),h_t,x_{t+1}))$
\EndFor
\State update slow parameters $\Theta$ by backpropagation through the accumulated selected read computations
\State \Return updated $\Theta$ and $\calS$
\end{algorithmic}
\end{algorithm}

\paragraph{Block-lagged approximation.}
For throughput, an implementation may instead hold $\calS$ fixed during the forward pass for a whole block, backpropagate through that block, and then apply detached \textsc{DGMObserve} updates for $t=1,\ldots,L$. This block-lagged DGM training is more parallel, but it is an approximation: token $t+1$ in the block does not see the state update produced after token $t$. It therefore introduces a train/inference mismatch and should not be described as the same observation program as streaming inference.

\subsection{Training modes for representation drift}

Anchors, centroids, and edges live in query space. If the encoder or query map changes rapidly while the DGM state persists, old anchors and edges describe stale geometry. Reports should therefore state which training mode is used.

\paragraph{Mode A: frozen encoder.}
Train the backbone and query map first, then freeze $\phi_\theta$, $Q$, and the output embedding used by DGM writes while running DGM runtime learning. This is the cleanest protocol for the first experiments and is the default interpretation of the theory.

\paragraph{Mode B: EMA query encoder.}
Maintain a slowly moving query encoder
\begin{align*}
\bar\theta\leftarrow \beta\bar\theta+(1-\beta)\theta,
\end{align*}
and create anchors, centroids, and edge generators in the query space of $\phi_{\bar\theta}$ and $Q_{\bar\theta}$. The backbone can still train, but the DGM state is tied to the slower geometry.

\paragraph{Mode C: periodic state rebuild.}
During end-to-end training, rebuild or clear the DGM state every $K$ optimization steps, or rebuild it from a replay buffer under the current query map. The formal guarantees then apply within a rebuild interval. Cross-interval state carryover is a systems approximation, not a claim of anchor-stable refinement.

\subsection{Non-canonical adapter extension}

The DGM-Embedding layer is the canonical layer because its target is the observed next-token embedding. A richer optional layer stores a local linear repair map, but it requires an explicit residual target and should not be conflated with the next-token memory mechanism:
\begin{align*}
\calS_t^{\mathrm{adp}}=\{a_j,c_j,W_j,n_j\}_{j=1}^{N_t}\cup G_t,
\qquad
W_j\in\R^{d\times r}.
\end{align*}
Let $K:\R^d\to\R^r$ be a learned or fixed key projection and set $k_t=Kh_t$. The adapter read is
\begin{align*}
\Delta h_t=\sum_{j\in N(q_t)}\alpha_j(q_t)W_jk_t,
\qquad
\tilde h_t=h_t+\gamma_\theta(h_t)\Delta h_t.
\end{align*}
Here $W_jk_t$ is a hidden-space residual, so the write rule requires an explicit residual target $v_t\in\R^d$. Natural choices include a token-embedding target $P E[x_{t+1}]$, a teacher-forced output-error target $P(E[x_{t+1}]-\hat e_t)$, or a distillation target $h_t^{\mathrm{teacher}}-h_t$. Without such a target, the adapter is only an architectural extension.

Given a local residual target, the projected rank-one repair for a selected prototype is the same residual-correction pattern that appears in passive-aggressive updates, RLS/ridge-style online least squares, and delta-rule fast-weight memories \citep{crammer2006online,vovk1997competitive,schlag2021linear}:
\begin{align*}
W_j^+=W_j+
\frac{\eta\alpha_j(q_t)}
{1+\eta\alpha_j(q_t)\norm{k_t}_2^2}
\big(v_t-W_jk_t\big)k_t^\top.
\end{align*}
Equivalently, a memory may maintain small ridge sufficient statistics and solve the local adapter in closed form \citep{tikhonov1977solutions,hoerl1970ridge}.

\subsection{Cost and systems notes}

During sequential detached training, backpropagation sees the selected read computation, while the DGM state is updated in the same order as streaming inference. The important comparison is with test-time gradient adaptation. A TTT-style layer typically performs an inner backward pass at inference time. A DGM layer performs retrieval, sparse local read, and closed-form or sufficient-statistic repair. The cost is forward-only:
\begin{align*}
\text{DGM inference}=\text{backbone forward}+\text{retrieval}+\text{sparse local repair}.
\end{align*}

\begin{theorem}[DGM layer cost]
\label{thm:dgm-layer-cost:informal}
Consider a DGM adapter extension with hidden dimension $d$, query dimension $d_q$, local rank $r$, candidate count $|N|$, selected incident edge-gate count $Q_N$, and prototype-bank query cost $C_{\mathrm{ret}}(N_t,d_q)$. For one token, the read overhead is
\begin{align*}
O(dd_q+C_{\mathrm{ret}}(N_t,d_q)+Q_N+|N|dr).
\end{align*}
If the write path is detached and uses rank-one local repair, the write overhead is
\begin{align*}
O(dd_q+C_{\mathrm{ret}}(N_t,d_q)+Q_N+|N|(d_q+dr)).
\end{align*}
For a block of length $L$, the DGM state history need not be stored for backpropagation; the additional activation memory is the selected read computation, not the full sequence of memory states. The DGM-Embedding layer replaces the $|N|dr$ adapter term by the selected embedding-logit cost.
\end{theorem}

The prototype bank can be sharded by the same principle used in retrieval and sparse expert systems. Each device owns a subset of prototypes. A query is routed to a small set of shards, each shard returns local top-$k$ candidates, and a final top-$k$ merge selects the memories used by the read path. Only selected memories receive gradients. Detached writes are sent back to the owning shards.

Budgeting is necessary for long training. A practical DGM layer maintains $N_t\le N_{\max}$, merges only compatible unprotected prototypes, and otherwise evicts or compresses low-score memories. These budget actions are maintenance operations, not learning operations in the formal refine-or-repair sense. The refine-or-repair theorems apply between maintenance events; merge, eviction, and compression may destroy information and must be evaluated as resource-management approximations.

\subsection{Closed-form local repair}

DGM-Ridge is the main closed-form adapter used in our checks. Let $Z\in\R^{n\times d}$ be normalized support embeddings stored by a DGM memory and let $Y\in\R^{n\times C}$ be one-hot consequences. The adapter computes the dual ridge/Tikhonov solution \citep{tikhonov1977solutions,hoerl1970ridge}
\begin{align*}
\Gamma=(ZZ^\top+\lambda I)^{-1}Y,
\qquad
\operatorname{logits}(z)=zZ^\top\Gamma.
\end{align*}
This is projection-style repair: it fits the local adapter in one linear solve rather than by iterated backpropagation. In a large model, the same idea can be applied per prototype, per document, or per retrieved shard after the relevant distinction structure has been fixed.
