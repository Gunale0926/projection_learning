\section{DGM Layers for Large-Model Training}
\label{sec:large-model-training}

The data structure above is useful as a runtime learner, but a large model needs a layer. The layer must satisfy two constraints that pull in opposite directions. Its read computation must be differentiable, so that the backbone can learn to use the memory. Its write computation must be cheap and stable, so that training and inference can use the same update rule without backpropagating through the entire memory history.

\subsection{A trainable DGM layer}

Let $h_t\in\R^d$ be the hidden state at a Transformer layer. A DGM layer maintains a prototype bank
\begin{align*}
\calS_t=\{c_j,A_j,B_j,n_j\}_{j=1}^{N_t},
\end{align*}
where $c_j\in\R^{d_q}$ is a prototype and $A_j\in\R^{r\times d}$, $B_j\in\R^{d\times r}$ define a rank-$r$ local adapter. The layer first computes a query
\begin{align*}
q_t=Qh_t.
\end{align*}
It retrieves a sparse neighborhood $N_k(q_t)$ and computes weights
\begin{align*}
\alpha_j(q_t)=\frac{\exp(-\norm{q_t-c_j}_2^2/T)}{\sum_{i\in N_k(q_t)}\exp(-\norm{q_t-c_i}_2^2/T)}.
\end{align*}
The read path is
\begin{align*}
\Delta h_t=\sum_{j\in N_k(q_t)}\alpha_j(q_t)B_jA_jh_t,
\qquad
\tilde h_t=h_t+\gamma_\theta(h_t)\Delta h_t.
\end{align*}
This is a sparse mixture of local low-rank repairs. It is differentiable through the selected memories, routing weights, and backbone parameters. The top-$k$ retrieval itself may be treated as a sparse selection operator, as in mixture-of-experts and retrieval systems.

\subsection{Detached write path}

After the training target is observed, the layer updates its fast state without placing the entire update history on the gradient tape. For language modeling, the target is $x_{t+1}$. A simple local write target is the next-token embedding
\begin{align*}
v_t=E[x_{t+1}].
\end{align*}
Each selected prototype can store a local Projection Memory $W_j$ or update its low-rank adapter using a projected local objective. In the simplest embedding-memory form,
\begin{align*}
r_j^+=\frac{n_jr_j+\alpha_j(q_t)v_t}{n_j+\alpha_j(q_t)},
\qquad
n_j^+=n_j+\alpha_j(q_t).
\end{align*}
In the richer adapter form, the local memory is repaired by a rank-one projection update. The update is detached:
\begin{align*}
\operatorname{stopgrad}(\calS_{t+1})=\operatorname{DGMObserve}(\operatorname{stopgrad}(\calS_t),h_t,x_{t+1}).
\end{align*}
Thus the model trains slow parameters through the read path and learns fast state through the same observation rule used at inference.

\begin{algorithm}[H]
\caption{\textsc{TrainStep} for a DGM Transformer layer}
\label{alg:dgm-train-step}
\begin{algorithmic}[1]
\State \textbf{Input:} token block $x_{1:L}$, slow parameters $\Theta$, DGM state $\calS$
\For{$t=1,\ldots,L$}
\State compute hidden state $h_t$ using the backbone
\State $q_t\gets Qh_t$
\State retrieve $N_k(q_t)$ from the prototype bank
\State read $\Delta h_t\gets\sum_{j\in N_k(q_t)}\alpha_j(q_t)B_jA_jh_t$
\State continue the network with $\tilde h_t=h_t+\gamma_\theta(h_t)\Delta h_t$
\EndFor
\State compute next-token loss $\calL(\Theta,\calS)$
\State update slow parameters $\Theta$ by backpropagation through the read path
\For{$t=1,\ldots,L$}
\State update $\calS$ by detached \textsc{DGMObserve} using consequence $x_{t+1}$
\EndFor
\State \Return updated $\Theta$ and $\calS$
\end{algorithmic}
\end{algorithm}

\subsection{Why this can scale}

The layer is not a replacement for backpropagation through the backbone. It is a replacement for inference-time and history-time backpropagation through the adaptive state. During training, backpropagation sees the selected read computation. During inference, the same layer performs forward retrieval and detached state repair. Therefore the train and inference programs differ only in whether slow parameters receive gradients.

\begin{theorem}[DGM layer cost, informal version of Theorem~\ref{thm:dgm-layer-cost:formal}]
\label{thm:dgm-layer-cost:informal}
Consider a DGM layer with hidden dimension $d$, query dimension $d_q$, rank $r$, top-$k$ retrieval, and prototype-bank query cost $C_{\mathrm{ret}}(N_t,d_q)$. For one token, the read overhead is
\begin{align*}
O(dd_q+C_{\mathrm{ret}}(N_t,d_q)+kdr).
\end{align*}
If the write path is detached and uses rank-one local repair, the write overhead is
\begin{align*}
O(dd_q+C_{\mathrm{ret}}(N_t,d_q)+k(d_q+dr)).
\end{align*}
For a block of length $L$, the DGM state history need not be stored for backpropagation; the additional activation memory is the selected read computation, not the full sequence of memory states.
\end{theorem}

The important comparison is with test-time gradient adaptation. A TTT-style layer typically performs an inner backward pass at inference time. A DGM layer performs retrieval, sparse local read, and closed-form or sufficient-statistic repair. The cost is forward-only:
\begin{align*}
\text{DGM inference}=\text{backbone forward}+\text{retrieval}+\text{sparse local repair}.
\end{align*}

\subsection{Distributed training}

The prototype bank can be sharded by the same principle used in retrieval and sparse expert systems. Each device owns a subset of prototypes. A query is routed to a small set of shards, each shard returns local top-$k$ candidates, and a final top-$k$ merge selects the memories used by the read path. Only selected memories receive gradients. Detached writes are sent back to the owning shards.

Budgeting is necessary for long training. A practical DGM layer maintains
\begin{align*}
N_t\le N_{\max}.
\end{align*}
When the bank is full, the layer may merge compatible prototypes, evict low-usage prototypes, or compress old prototypes into a shared local memory. These operations are modeling choices rather than new learning principles. The principle remains refine when the representation is too coarse, and repair when the local state is adequate.

\subsection{Closed-form adapters}

The local memory need not be an exemplar table. A particularly useful large-model adapter is a closed-form ridge readout over the retrieved support memory. Let $Z\in\R^{n\times d}$ be normalized support embeddings stored by a DGM memory and let $Y\in\R^{n\times C}$ be one-hot consequences. The DGM-Ridge adapter computes
\begin{align*}
\Gamma=(ZZ^\top+\lambda I)^{-1}Y,
\qquad
\operatorname{logits}(z)=zZ^\top\Gamma.
\end{align*}
This is a projection-style repair: it fits the local adapter in one linear solve rather than by iterated backpropagation. When $n$ is small, as in few-shot adaptation or a retrieved local neighborhood, the solve is cheap. In a large model, the same idea can be applied per prototype, per document, or per retrieved shard.
