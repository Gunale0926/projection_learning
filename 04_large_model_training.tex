\section{DGM-Embedding: A Runtime Memory Layer for Language Models}
\label{sec:large-model-training}

For language modeling, the world state is a prefix and the consequence is the next observed token. This makes the refine-or-repair decision concrete. If a prefix type is already represented, the layer should repair its next-token memory. If the same runtime concept is repeatedly asked to predict incompatible next-token consequences, the layer should refine the concept graph. DGM-Embedding is the canonical large-model layer because its detached target is observed without an inner optimization step: the next-token embedding.

The layer is the language-model analogue of the finite DGM loop. The prefix hidden state chooses edge-gated concepts; their memories produce a sparse logit correction; the true next token then supplies the detached consequence used for repair or refinement. There is no inference-time inner optimization problem in this write path.

\subsection{Hidden-state concepts}

Let a frozen or slowly trained backbone produce hidden states
\begin{align*}
h_t=\phi_\theta(x_{\le t}),
\end{align*}
and let
\begin{align*}
q_t=Qh_t\in\R^{d_q}
\end{align*}
be the DGM query. A DGM layer maintains frozen query-space anchors $a_j\in\R^{d_q}$, movable query-space centroids $c_j\in\R^{d_q}$, embedding memories $r_j\in\R^{d_e}$, counts $n_j$, and distinction edges:
\begin{align*}
\calS_t^{\mathrm{emb}}
=
\left(\{a_j,c_j,r_j,n_j\}_{j=1}^{N_t},G_t\right).
\end{align*}
The formal information structure is generated by anchors and edges. Retrieval uses centroids as adaptive statistics, while edge generators are computed from anchors. The recall-oriented candidate set $N(q_t)$ is the same one used by DGM: centroid neighbors for adaptive locality, anchor neighbors for formal route recall, and graph neighbors for recent conflicts.

\subsection{Logit correction}

The read path is differentiable through the selected memories, routing weights, and backbone parameters. With output embedding $e_y$, the DGM read contributes a sparse logit correction
\begin{align*}
\Delta\ell_t(y)=\sum_{j\in N(q_t)}\alpha_j(q_t)e_y^\top r_j,
\end{align*}
where $\alpha_j$ is edge-gated as in \eqref{eq:edge-gated-routing}. Equivalently, define the mixed memory vector
\begin{align*}
r_{\mathrm{mix}}(q_t)=\sum_{j\in N(q_t)}\alpha_j(q_t)r_j.
\end{align*}
Then $\Delta\ell_t(y)=e_y^\top r_{\mathrm{mix}}(q_t)$, and the final logits are
\begin{align*}
\ell(y\mid x_{\le t})=\ell_\theta(y\mid x_{\le t})+\lambda_t\Delta\ell_t(y).
\end{align*}
Here $\lambda_t$ is a learned or fixed trust weight. With tied output embeddings and $d_e=d$, the same correction can be implemented as
\begin{align*}
h_t^{\mathrm{DGM}}=h_t+\lambda_t r_{\mathrm{mix}}(q_t),
\end{align*}
followed by the usual LM-head multiplication. This avoids an extra vocabulary-sized projection.

\subsection{Read before observe}

DGM-Embedding uses the same streaming order during training and inference whenever external consequences are available:
\begin{align*}
\text{predict }x_{t+1}\rightarrow\text{observe }x_{t+1}\rightarrow\text{repair or refine}\rightarrow\text{predict }x_{t+2}.
\end{align*}
The order is important: the layer must predict before it sees the token that will update memory. After the next token is observed, $v_t=e_{x_{t+1}}$ gives the detached sufficient-statistic write for every selected $j\in N(q_t)$, with all unselected memories unchanged:
\begin{align*}
r_j^+=\frac{n_jr_j+\alpha_j(q_t)v_t}{n_j+\alpha_j(q_t)},
\qquad
n_j^+=n_j+\alpha_j(q_t).
\end{align*}
Hard-update mode is the special case $\alpha_{j^\star}=1$ and $\alpha_j=0$ for $j\ne j^\star$. If the penalized refinement test in \eqref{eq:refine-accept} accepts a new concept, the layer creates a new anchor and centroid at $q_t$, initializes $r_{\mathrm{new}}=v_t$, and adds distinction edges to $\operatorname{Confused}_q(q_t,x_{t+1})$. Ordinary repairs may update centroids and memories but leave anchors fixed.

During streaming prediction, document continuation, user correction, tool feedback, or supervised next-token evaluation, the next observed token is external evidence and can refine or repair the runtime memory. During unconstrained generation, the model should not update from its own sampled token by default, because self-generated samples are actions, not observed consequences.

\begin{algorithm}[H]
\caption{\textsc{ObserveNextToken} for DGM-Embedding}
\label{alg:dgm-lm}
\begin{algorithmic}[1]
\State \textbf{Input:} prefix $x_{\le t}$, observed token $x_{t+1}$, DGM state $\calS_t$
\State $h_t\gets\phi_\theta(x_{\le t})$ and $q_t\gets Qh_t$
\State retrieve $N(q_t)$ and compute edge-gated logits $\ell_\theta+\lambda_t\Delta\ell_t$
\State $j^\star\gets$ highest-weight concept
\State $\rho\gets-\log p(x_{t+1}\mid x_{\le t})$
\If{$\rho\le\tau_{\mathrm{repair}}$}
\State update DGM-Embedding memories for all $j\in N(q_t)$ using soft credit $\alpha_j(q_t)$
\Else
\State accept a new anchored concept only if the penalized refinement gain test holds
\EndIf
\State \textbf{Hard variant:} set $\alpha_{j^\star}=1$ and $\alpha_j=0$ for $j\ne j^\star$
\State \Return updated state
\end{algorithmic}
\end{algorithm}

\subsection{Sequential detached training}

The update is detached:
\begin{align*}
\operatorname{stopgrad}(\calS_{t+1})=\operatorname{DGMObserve}(\operatorname{stopgrad}(\calS_t),h_t,x_{t+1}).
\end{align*}
In sequential detached training, the model trains slow parameters through the read path and learns fast state through the same observation rule used at inference. The layer is not a replacement for backpropagation through the backbone. It is a replacement for inference-time and history-time backpropagation through the adaptive state.

\begin{algorithm}[H]
\caption{\textsc{SequentialDetachedTrainStep} for a DGM-Embedding Transformer layer}
\label{alg:dgm-train-step}
\begin{algorithmic}[1]
\State \textbf{Input:} token block $x_{1:L}$, slow parameters $\Theta$, DGM state $\calS$
\State $\calL\gets0$
\For{$t=1,\ldots,L$}
\State compute hidden state $h_t$ using the backbone
\State $q_t\gets Qh_t$
\State retrieve $N(q_t)$ from the concept bank
\State compute edge-gated weights $\alpha_j(q_t)$
\State apply the DGM-Embedding read
\State produce logits for $x_{t+1}$ and accumulate $\calL\gets\calL+\ell_t$
\State $\calS\gets\operatorname{stopgrad}(\textsc{DGMObserve}(\operatorname{stopgrad}(\calS),h_t,x_{t+1}))$
\EndFor
\State update slow parameters $\Theta$ by backpropagation through the accumulated selected read computations
\State \Return updated $\Theta$ and $\calS$
\end{algorithmic}
\end{algorithm}

\paragraph{Block-lagged approximation.}
For throughput, an implementation may instead hold $\calS$ fixed during the forward pass for a whole block, backpropagate through that block, and then apply detached \textsc{DGMObserve} updates for $t=1,\ldots,L$. This block-lagged DGM training is more parallel, but it is an approximation: token $t+1$ in the block does not see the state update produced after token $t$. It therefore introduces a train/inference mismatch and should not be described as the same observation program as streaming inference.

The main architectural claim stops here: DGM-Embedding is a next-token memory layer with differentiable reads and detached writes. Training modes for representation drift, the optional adapter variant, systems notes, and closed-form ridge repair are implementation extensions rather than the paper's central object; we give them in Appendix~\ref{app:dgm-extensions}.
