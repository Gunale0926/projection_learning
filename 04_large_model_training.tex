\section{CGA-LM: Replacing Token Attention with Concept Graph Attention}
\label{sec:large-model-training}

For language modeling, the world state is a prefix and the consequence is the
next token.
CGA-LM replaces the causal attention sublayer inside a decoder block with
Concept Graph Attention:
\[
\operatorname{CGABlock}(X)
=
X+\operatorname{CGA}(\operatorname{Norm}(X)).
\]
The block has the same external shape contract as attention,
$[B,L,d_{\rm model}]\mapsto[B,L,d_{\rm model}]$.
The surrounding MLP, residual stream, layer normalization, embeddings, and
language-model head are otherwise standard decoder components.

\subsection{Causal concept attention}

At layer $\ell$, token $t$ produces read query, write key, and write value
projections:
\[
q_t=Q_\ell x_t,\qquad k_t=K_\ell x_t,\qquad v_t=V_\ell x_t.
\]
The layer-local CGA state is a graph
\[
\calG_t^\ell
=
\left(
\{a_j,c_j,m_j,n_j\}_{j=1}^{N_t},
E_t
\right).
\]
The read is causal: token $t$ reads only concepts created from earlier tokens.
For active concepts,
\[
\alpha_{tj}
=
\operatorname{softmax}_{j\le N_t}
\left(\gamma_\ell q_t^\top c_j\right),
\qquad
z_t=\sum_{j\le N_t}\alpha_{tj}m_j,
\qquad
y_t=O_\ell z_t .
\]
The scale $\gamma_\ell$ is learned.
Multi-head CGA uses independent address and value subspaces per head and
concatenates the head outputs before the output projection, exactly as
multi-head attention does.

\subsection{Read before write}

The update order is the same during training and autoregressive inference:
\[
\text{read from }\calG_t^\ell
\rightarrow
\text{emit }y_t
\rightarrow
\text{write }(k_t,v_t)
\rightarrow
\text{repair or refine }\calG_{t+1}^\ell .
\]
This prevents a token from reading its own write.
A repair updates an existing concept's sufficient statistics.
A refinement creates a new concept and optional distinction edges.
The MQAR implementation uses delayed writes so that a key token can bind to the
following value token; language-model variants can use the same default or a
learned write gate.

\subsection{Training boundary}

Slow neural parameters are trained by ordinary next-token backpropagation
through the CGA read path.
The graph mutation itself is detached:
\[
\calG_{t+1}^\ell
=
\operatorname{CGAWrite}
\left(
\operatorname{stopgrad}(\calG_t^\ell),
\operatorname{stopgrad}(k_t),
v_t
\right).
\]
Thus CGA-LM is not a backprop-free language model.
It is an attention replacement whose fast state is updated by a forward-time
concept-graph rule while the slow projections are trained end-to-end.

\begin{algorithm}[H]
\caption{\textsc{CGALayerForward}}
\label{alg:cga-lm}
\begin{algorithmic}[1]
\State \textbf{Input:} hidden states $x_{1:L}$, empty concept graph $\calG$
\For{causal blocks}
\State project block queries, write keys, and values
\State read all active concepts with continuous attention weights
\State emit block outputs through the output projection
\State form delayed or gated write pairs
\State assign writes to compatible concepts or proposed new concepts
\State scatter-add sufficient statistics and commit graph updates
\EndFor
\State \Return contextualized hidden states
\end{algorithmic}
\end{algorithm}

\subsection{Scaling path}

The reference CGA implementation uses dense concept reads.
This is appropriate for state-equalized MQAR where the concept budget is small.
For large concept graphs, sparse retrieval, fused dense-read kernels,
budget-aware merge and eviction, and sharded concept banks are systems work.
The paper therefore separates the architectural claim from the kernel claim:
CGA changes the memory basis of attention from tokens to concepts; optimized
implementations should then choose dense or sparse concept attention depending
on the concept budget.
