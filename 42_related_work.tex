\section{Related Work}
\label{app:related}

The relevant comparison is not whether CA uses centers, memory, routing, or online updates. All of these ingredients have long histories. The question is whether a runtime learner distinguishes two operations that are often collapsed: repairing state inside the current representation and refining the representation when consequences reveal a missing distinction.

\subsection{Methods that repair without refining}

Cache language models and $k$NN-LMs store or retrieve context-token examples at inference time \citep{grave2017improving,khandelwal2020generalization}. Memorizing Transformers, RETRO, RAG, and sparse retrieval systems extend this idea with larger or external stores \citep{wu2022memorizing,borgeaud2022improving,lewis2020retrieval,rae2016scaling}. These methods make past examples available, but the default response to surprise is more storage or retrieval, not the creation of a new runtime concept forced by consequence conflict.

Fast weights, linear attention memories, delta-rule updates, passive-aggressive learning, online ridge/RLS, and Projection Memory repair associative state inside a fixed geometry \citep{ba2016using,katharopoulos2020transformers,schlag2021linear,crammer2006online,vovk1997competitive,bregman1967relaxation,tikhonov1977solutions,hoerl1970ridge}. Adapter modules and LoRA provide parameter-efficient adaptation state in large pretrained models \citep{houlsby2019parameter,hu2022lora}. Conservative Projection Memory (CPM) is especially close to our repair side because it detects zero innovation with nonzero residual as a conflict. CA uses this diagnosis differently: a zero-innovation conflict is evidence that the current information cell is too coarse, so projection-style repair should stop until the information structure is refined.

Test-time adaptation and test-time training update model state or parameters by optimizing an inference-time objective \citep{sun2020test,wang2021tent,niu2022eata,zhang2022memo,sun2025learning}. These methods ask what surrogate loss should be optimized at test time. CA asks a prior structural question: is the error repairable under the current distinctions, or does it reveal a missing distinction?

These methods solve the repair-gap side of runtime learning. They do not, by themselves, provide an explicit consequence-forced refinement operation.

\subsection{Methods that refine without local repair}

Decision trees, online trees, Hoeffding trees, adaptive trees, extremely fast decision trees, and Mondrian forests refine leaves using information gain, variance reduction, or streaming split tests \citep{quinlan1986induction,breiman1984classification,domingos2000mining,bifet2009adaptive,manapragada2018extremely,lakshminarayanan2014mondrian}. These are the closest algorithmic relatives of CA's refinement test. The difference is structural: trees store distinctions in a rooted hierarchy, while CA stores local pairwise distinction edges between runtime concepts and attaches repairable memories to the resulting nodes.

Adaptive Resonance Theory, including ART 1 for binary patterns and ART 2 for analog input patterns, addresses stable incremental category creation under mismatch and vigilance \citep{carpenter1987massively,carpenter1987art2,grossberg2013adaptive}. Prototype methods, LVQ, SOMs, neural gas, growing neural gas, and Prototypical Networks represent regions or classes by stored centers or exemplars \citep{kohonen1990self,kohonen1995self,martinetz1991neural,fritzke1995growing,snell2017prototypical}. CA shares the vocabulary of category creation and prototypes, but its concepts are operational hidden-state concepts whose edges are consequence-forced binary generators. The node is not only a region; it is a place where local memory can be repaired after the distinction is accepted.

Concept Bottleneck Models and Formal Concept Analysis emphasize explicit concept structures \citep{koh2020concept,ganter1999formal}. CA does not claim human-interpretable concepts by default. Its concepts are runtime concepts: hidden-state anchors, centroids, local memories, and pairwise distinctions created from prediction consequences.

These methods solve the refinement half in various forms. They typically do not pair refinement with graph-gated neural local memories that continue to be repaired after the split.

\subsection{Methods that route without runtime concept creation}

Mixture-of-experts, Switch Transformers, and Product-Key Memory route tokens or queries to sparse trained capacity \citep{shazeer2017outrageously,fedus2022switch,lample2019large}. Memory Networks, End-to-End Memory Networks, Neural Turing Machines, and Differentiable Neural Computers learn external read/write controllers \citep{weston2015memory,sukhbaatar2015end,graves2014neural,graves2016hybrid}. These systems are important precedents for sparse reads and external memory. Fixed-bank CA differs in write semantics: the routed objects are fixed concept identities whose runtime value memories are repaired by the observed prefix. The graph-growing extension goes further by creating runtime concepts and edges whose predicates generate an information structure.

Continual learning studies forgetting, replay, regularization, and capacity growth across task sequences \citep{kirkpatrick2017overcoming,lopezpaz2017gradient,rusu2016progressive,yoon2018lifelong}. CA is complementary: it is an instance-level runtime data structure for local concept growth and memory repair, not a complete task-sequence continual learning system.

\subsection{CA's position}

CA combines refinement and repair but keeps them distinct:
\begin{align*}
\Delta_{\mathrm{refine}}\text{ dominates}\Rightarrow\text{refine},
\qquad
\Delta_{\mathrm{repair}}\text{ dominates}\Rightarrow\text{repair}.
\end{align*}
The measure-theoretic formalism follows standard sigma-algebra and conditional-expectation language \citep{billingsley1995probability,kallenberg2002foundations}. The Bayes-risk order is related to Blackwell comparison and statistical decision theory \citep{blackwell1953equivalent,berger1985statistical}. The log-loss and squared-loss value identities are standard information-theoretic and Hilbert-space facts \citep{cover2006elements,gneiting2007strictly}. The contribution is to place these facts inside a runtime-learning model in which some errors are repair signals and others are evidence of missing distinctions.
