\section{Related Work}
\label{app:related}

\paragraph{Prototype and memory methods.} Prototype and memory methods represent data by stored exemplars or centers, including nearest-neighbor classifiers, LVQ, SOMs, and neural gas \citep{kohonen1990self,martinetz1991neural}. DGM uses the same computational intuition but adds a refinement rule and an information-structure semantics. It is not merely a prototype model: prototypes define runtime concepts, local memories repair within concepts, and distinction edges are consequence-forced binary generators that both define atoms and gate routing.

\paragraph{Decision trees and splitting criteria.} Classical decision trees use information gain and variance reduction as splitting criteria, and online decision trees such as Hoeffding trees make greedy streaming split decisions \citep{domingos2000mining}. In our view, a decision tree is a hierarchical hard-routing special case of a distinction graph. It stores distinctions in a rooted hierarchy rather than as local graph edges between concepts.

\paragraph{Sparse experts and retrieval layers.} Mixture-of-experts and retrieval-augmented models scale neural computation by selecting a small subset of parameters or memories for each input \citep{shazeer2017outrageously,lample2019large}. A DGM layer has a similar sparse read path, but the selected objects are runtime concepts with local memories, edge-generated information structure, and a detached write path that can refine or repair after observing consequences.

\paragraph{Cache and nonparametric language models.} Cache language models and $k$NN language models improve next-token prediction by retrieving recent or datastore examples \citep{grave2017improving,khandelwal2020generalization}. DGM-Mean is close in spirit to this family, but its memory is organized by graph-induced concepts rather than by an undifferentiated datastore, and surprising consequences can create new distinctions rather than only update retrieval statistics.

\paragraph{Memory-augmented networks.} Memory networks, fast weights, and dynamically expanded architectures add explicit state or capacity to neural models \citep{weston2015memory,ba2016using,rusu2016progressive}. DGM's distinction is the refine-or-repair decomposition: expansion is accepted only when it decreases a penalized local objective, while ordinary state error is handled by local repair.

\paragraph{Information theory and statistical decision theory.} The log-loss identity is the standard relation between optimal code length and conditional entropy, while the squared-loss identity is the orthogonal decomposition of conditional expectations in $L^2$. Our contribution is to place these identities inside a runtime-learning story: the value of learning a distinction is its reduction in Bayes risk.

\paragraph{Backpropagation and gradient learning.} Backpropagation supplies differentiable credit assignment for fixed computation graphs \citep{rumelhart1986learning}. Distinction Learning addresses a different failure mode: the current information structure may not contain the distinctions needed for any measurable predictor to succeed.

\paragraph{Projection learning and online repair.} Bregman projection, mirror descent, proximal methods, passive-aggressive updates, and online ridge-style methods provide minimal-change state updates \citep{bregman1967relaxation,zinkevich2003online,crammer2006online,shalev2012online,hazan2016introduction,vovk1997competitive,azoury2001relative}. Conservative Projection Memory uses this idea for runtime associative repair. The present paper demotes projection repair to the continuous-state layer that operates after the relevant information structure has been fixed.

\paragraph{Test-time adaptation.} Test-time training adapts parameters using auxiliary losses at inference time \citep{sun2020test}. TTT layers treat hidden states as trainable models updated on test sequences \citep{sun2025learning}. Distinction Learning suggests a complementary diagnostic: when runtime evidence conflicts inside an indistinguishable atom, the issue is not only how to update fast state, but what distinction the state failed to represent.
