\section{Conclusion}
\label{sec:conclusion}

Not every error is a gradient. Some errors are stale-state errors, and some are missing-distinction errors. Distinction Graph Machines turn this separation into an explicit runtime learner with anchored concepts, local memories, distinction edges, repair rules, and a train/test API. The guiding invariant is simple: refine when the atom is wrong, repair when the local state is wrong.

The theory explains why this separation matters. Contradictions inside an atom cannot be solved by optimizing a measurable predictor. They require refinement. Once the local concept is adequate, its state can be repaired by sufficient-statistic, ridge, or projection-memory updates. DGM turns that separation into a concrete runtime state: concepts hold local memories, and distinction edges generate the information structure that gates the read path.

The resulting picture places gradient learning, projection repair, and distinction refinement in one hierarchy. Backpropagation moves slow parameters; projection memory repairs fast state; DGM adds the missing operation of changing the information structure on which those states are defined. In language modeling, this means that a runtime learner need not backpropagate through the entire model at inference time: it can maintain hidden-state concepts, repair next-token memories, and create new distinctions when the stream reveals a genuinely new context.
