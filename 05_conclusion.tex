\section{Conclusion}
\label{sec:conclusion}

Runtime learning fails when we repair errors that actually require new distinctions. The repair/refinement gap decomposition makes this precise: some excess risk can be removed by changing the predictor at fixed information, and some can only be removed by changing the information itself. Distinction Graph Machines turn this separation into an explicit runtime learner with anchored concepts, local memories, distinction edges, and a read-before-observe update rule.

The theory explains why this separation matters. Contradictions inside an information cell cannot be solved by optimizing a predictor that still factors through the same information map. They require refinement. Once the local concept is adequate, its state can be repaired by sufficient-statistic, ridge, or projection-memory updates. DGM turns that separation into a concrete runtime state: concepts hold local memories, and distinction edges generate the information structure that gates the read path.

The resulting picture places gradient learning, local repair, and distinction refinement in one hierarchy. Backpropagation moves slow parameters; local memory repairs fast state; DGM adds the missing operation of changing the information structure on which those states are defined. In language modeling, this means that a runtime learner need not backpropagate through the entire model at inference time: it can maintain hidden-state concepts, repair next-token memories, and create new distinctions when the stream reveals a genuinely new context.
