\section{Introduction}
\label{sec:main-intro}

Runtime learning mechanisms usually respond to surprise by repairing state: cache a new example, update a local statistic, or adapt a small set of parameters.
These updates are powerful only if the current runtime representation already separates the cases that require different actions.
If two cases are indistinguishable to the runtime state but require different predictions, no repair of values inside that state can separate them.
This is a common failure mode in online systems with memory: a cache can store more examples, an adapter can move its parameters, and a local statistic can become well estimated, but all of these updates still assign one decision to one runtime cell if the representation has collapsed the relevant cases.

We model this failure using information partitions, following the decision-theoretic view that available information constrains admissible decision rules \citep{blackwell1953equivalent,berger1985statistical}.
A runtime state induces an equivalence relation over cases.
A repair update changes the payload assigned to an existing equivalence class; a refinement update changes the equivalence relation itself by adding an observable distinction.
In the simplest log-loss example, $\Omega=\{0,1\}$, $Y=\omega$, and $P(0)=P(1)=1/2$.
The trivial partition $\Pi=\{\{0,1\}\}$ has $R_{\log}(\Pi)=\log 2$, while the refined partition $\Pi^\star=\{\{0\},\{1\}\}$ has $R_{\log}(\Pi^\star)=0$.
No repair of the single atom can assign two different predictions.

The central decomposition is
\[
R_L(f_t)-R_L(\Pi^\star)
=
\underbrace{R_L(f_t)-R_L(\Pi_t)}_{\Delta_{\rm repair}}
+
\underbrace{R_L(\Pi_t)-R_L(\Pi^\star)}_{\Delta_{\rm refine}}.
\]
The first term is what better payloads, memories, or statistics can fix at the current partition.
The second is what can only be removed by changing the partition.
This decomposition is the organizing principle of the paper: surprise is not by itself evidence for refinement, because stale or noisy local statistics also cause error; refinement is justified only when the held-out consequence value of a new distinction exceeds what repair can explain.

This paper makes three contributions.
\begin{itemize}
    \item \textbf{Obstruction.} If predictions remain measurable with respect to a fixed partition, a positive refinement gap gives linear regret against a refined oracle.
    \item \textbf{Value.} Useful distinctions are consequence-sensitive: under uniqueness of Bayes actions, a refinement has zero value exactly when the refined Bayes action was already measurable under the old information; under log loss the value of adding $U$ is $I(Y;U\mid\Pi)$.
    \item \textbf{Implementation.} We instantiate the decision as Distinction Graph Machines (DGM), a graph-memory data structure whose accepted edges are distinctions and whose nodes carry repairable local memories.
\end{itemize}
Experiments are organized around the runtime-layer claim: on CIFAR-10 we compare DGM as a test-time graph-memory layer against a frozen source model, entropy minimization, a low-rank adapter baseline, and a TTT-style rotation-adaptation baseline.
The older controlled diagnostics are retained only as proof-of-mechanism checks outside the main empirical claim.

We do not claim that DGM dominates unrestricted end-to-end representation learning.
The obstruction applies to updates that preserve the relevant information map; representation-changing gradient updates are refinements in this model.
DGM is one guarded implementation of distinction acceptance under an explicit runtime information structure.
