\section{Introduction}
\label{sec:main-intro}

Attention can be read as a memory operation.
Given a prefix, a Transformer stores one key-value entry per token and later
queries retrieve a weighted combination of those token values.
This token table is powerful because the read is continuous and
differentiable, but it fixes the unit of memory: the model attends to token
events.
Long-context architectures have explored recurrent states, state-space models,
linear memories, sparse attention, and inference-time optimizers.
These approaches change the systems tradeoff, but the central memory question
remains: can the model attend over a smaller set of learned runtime concepts
instead of attending over every previous token?

We propose \emph{Concept Graph Attention} (CGA).
CGA keeps the familiar attention read,
\[
\operatorname{read}(q_t)=
\sum_{j=1}^{C}\operatorname{softmax}_j(s(q_t,c_{tj}))\,m_{tj},
\]
but changes the objects being read.
Instead of a token-indexed key-value table, each layer maintains a fixed bank
of $C$ concept slots.
Each slot has an address $c_{tj}$, value memory $m_{tj}$, and count statistics
that evolve causally over the prefix.
The read is attention over concepts; the write is a gated
sufficient-statistic update into the concept bank.
Thus the layer is attention-like in its readout but recurrent-memory-like in
its state:
\[
\begin{aligned}
\text{Transformer: tokens}&\rightarrow\text{attention over token memory},\\
\text{CGA: tokens}&\rightarrow\text{fixed concept memory}
\rightarrow\text{attention over concepts}.
\end{aligned}
\]

The fixed-bank design is deliberate.
In language modeling and long synthetic recall, the data stream is effectively
unbounded; a useful memory layer should eventually use its available capacity.
CGA makes the capacity explicit from the first step and focuses the model on
the harder problem:
which concepts should be read and how should each token update them?
The number of memory slots is a concept budget $C$ rather than the context
length $T$, giving a dense read cost of $O(TC)$ when $C\ll T$.
At a fixed runtime-state budget, CGA can trade the number of concept slots
against the per-slot value width.

This gives a different answer to the same question that motivates
test-time-training sequence layers \citep{sun2025learning}.
TTT layers make the hidden state a small model and update that model by a
self-supervised learning step on the test sequence.
CGA instead makes the hidden state a concept-indexed memory and updates that
memory by a forward-only sufficient-statistic rule.
Both mechanisms let a model adapt to information observed at inference time.
The difference is the object being learned:
\[
\begin{array}{ll}
\text{TTT:} & \text{test-time learning of hidden model weights},\\
\text{CGA:} & \text{test-time learning of concept-memory contents}.
\end{array}
\]
We use ``learning'' here in the operational stateful sense: future predictions
condition on a state estimated from the observed prefix.
CGA does not update the global model parameters during inference and does not
run an inference-time backward pass.

This paper makes three contributions.
\begin{itemize}
    \item \textbf{Architecture.} We introduce fixed-bank Concept Graph
    Attention, a causal mixer with test-time concept learning: it reads
    continuously from concept slots and updates them with gated writes and
    trainable retention decay while model weights stay fixed at inference time.
    \item \textbf{Theory.} We formalize CGA as attention over an online
    sufficient-statistic memory, specifying the state space, causal read/write
    equations, state-size scaling, scan-friendly update structure, and
    backpropagation boundary needed to compare it to recurrent, attention-like,
    and TTT-style sequence mixers.
    \item \textbf{Evaluation.} We implement CGA in the PoST/Zoology framework
    and evaluate it as an attention-like sequence mixer on state-equalized
    multi-query associative recall, with small language-model experiments as
    the scaling target.
\end{itemize}

The claim boundary is important.
CGA is not presented as a fully optimized replacement for every attention
kernel.
It is a method proposal for changing the memory basis of attention from tokens
to runtime concepts.
The current implementation trains slow projections by backpropagation, while
the concept-memory update itself is a forward-time operation.
This places CGA between attention, external memory, and efficient recurrent
mixers: it uses differentiable content-addressed reads like attention, but the
state being read is a compact, updateable concept memory rather than the entire
prefix token table or a set of test-time-trained hidden weights.
