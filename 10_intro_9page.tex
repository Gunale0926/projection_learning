\section{Introduction}
\label{sec:main-intro}

We study a basic obstruction in online prediction with mutable memory. A runtime state is not merely a vector; it is an information structure determining which histories remain distinguishable. Once two cases induce the same state, every predictor that factors through that state must give them the same action. More computation, better local statistics, or a more elaborate optimizer can repair the action assigned to the state, but cannot use a distinction that the state has erased.

The paper formalizes this point independently of architecture. A runtime representation induces a partition $\Pi$ of the case space. A \emph{repair} update changes the payload assigned to atoms of $\Pi$ while preserving the partition. A \emph{refinement} update joins $\Pi$ with a prediction-time observable distinction $U$, producing the finer partition $\Pi\vee U$. The question is therefore not whether a model updates at test time, but whether the update changes the events on which future predictions may condition.

The first result is an algorithm-independent obstruction. For any predictor $f\in\Pred(\Pi)$ and any refined comparator $\Pi^\star$,
\begin{align*}
R_L(f)-R_L(\Pi^\star)
=
\underbrace{R_L(f)-R_L(\Pi)}_{\text{repair gap}}
+
\underbrace{R_L(\Pi)-R_L(\Pi^\star)}_{\text{refinement gap}}.
\end{align*}
The first term can be attacked by estimation, optimization, caching, or local adaptation at fixed information. The second cannot. If every prediction remains $\Pi$-measurable, expected regret against a refined oracle is at least $T(R_L(\Pi)-R_L(\Pi^\star))$, up to a one-time refinement cost. Missing distinctions create linear regret not because the optimizer is weak, but because the admissible decision class is too coarse.

The second result identifies the unit of structural growth. A candidate bit is not valuable because it is balanced, geometrically salient, or frequent. It is valuable exactly to the extent that it changes the Bayes action. The minimal refinement is the join operation, and its value is the Bayes-risk decrease
\begin{align*}
V_L(U\mid\Pi)=R_L(\Pi)-R_L(\Pi\vee U).
\end{align*}
For log loss this is $I(Y;U\mid\Pi)$, and for squared loss it is $\E\|\E[Y\mid\Pi,U]-\E[Y\mid\Pi]\|_2^2$. Under uniqueness, a refinement has zero value iff the refined Bayes action already has an old-measurable version. This rules out accepting distinctions merely because they separate inputs rather than consequences.

The third result is algorithmic. Population value is not directly observable, so refinement must be guarded. We give a sample-split rule: score repair and refinement candidates on a held-out buffer, charge structural cost, and accept refinement only when its penalized empirical gain beats the best repair gain by a uniform concentration margin. This is not a global training theorem; it is the statistical contract needed for a single adaptive structural update to mean what the theory says it means.

The theory leads us to propose Conceptual Attention (CA). A concept is an operational object: a persistent address for a prediction-relevant information cell, not necessarily a human-semantic category. The general DGM picture allows online concept and edge growth, but language-model scale requires a controlled architectural test. We therefore evaluate fixed-bank CA: each layer preallocates a bounded set of fixed random concept identities and maintains only a mutable concept-value memory at runtime. Tokens read with attention over concept identities and write by gated continuous updates with trainable retention. For parallel training and efficient CUDA execution, the implementation is block-causal: all tokens in a block read the entering memory, and their writes are accumulated into the next block state. Block size one recovers the token-streaming rule; larger blocks are an explicit systems approximation, not a change to the information-theoretic obstruction.

This separation gives the paper's contribution structure. Sections~\ref{sec:main-setup}--\ref{sec:main-guard} are the theory: the repair-only obstruction, the value characterization, and the guarded refinement test. Section~\ref{sec:main-ca} presents the fixed-bank CA layer actually used for the main experiments, including its block update and bounded-state accounting. Section~\ref{sec:main-experiments} evaluates this proposal on state-equalized MQAR and matched language-model pretraining. The appendix contains the general graph-growing CA extension with online edge refinement, finite-class oracle inequalities, data-structure bounds, projection-memory bridges, kernel/scaling notes, and full proofs.
