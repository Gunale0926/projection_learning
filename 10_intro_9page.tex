\section{Introduction}
\label{sec:main-intro}

Attention can be read as a memory operation.
Given a prefix, a Transformer stores one key-value entry per token and later
queries retrieve a weighted combination of those token values.
This token table is powerful because the read is continuous and differentiable,
but it also fixes the unit of memory: the model attends to token events.
Long-context architectures have therefore explored many ways to replace or
compress token attention, including recurrent states, state-space models,
linear memories, and inference-time optimizers.
These approaches change the systems tradeoff, but they do not directly expose
the structural question: should a runtime sequence model store another token,
repair an existing memory cell, or create a new latent concept?

We propose \emph{Concept Graph Attention} (CGA).
CGA keeps the familiar attention read,
\[
\operatorname{read}(q)=\sum_{j\le N_t}
\operatorname{softmax}_j(s(q,c_j))\,v_j,
\]
but changes the objects being read.
Instead of attending over raw token keys and values, CGA attends over a runtime
graph of concepts.
Each concept has an address, a value memory, sufficient statistics, and optional
distinction edges to other concepts.
As tokens arrive, the graph is updated by local write rules: compatible
observations repair an existing concept, old concept statistics decay through
trainable retention factors, and incompatible observations create or refine
concepts.
Thus the layer is attention-like in its readout but graph-memory-like in its
state:
\[
\text{Transformer: tokens}\rightarrow\text{attention over token memory},
\qquad
\text{CGA: tokens}\rightarrow\text{concept graph}\rightarrow
\text{attention over concepts}.
\]

This positioning is deliberate.
We do not claim that CGA removes attention; rather, it changes what attention is
computed over.
The potential advantage is that the number of memory slots is a concept budget
$C$ rather than the sequence length $T$, giving a read cost of $O(TC)$ when
$C\ll T$.
More importantly, the memory slots are updateable runtime objects.
They can aggregate repeated evidence, preserve key-value bindings, forget stale
context through learned retention, and carry local distinction structure across
blocks.
This makes CGA a candidate drop-in replacement for attention variants whose
read can be interpreted as content-addressed retrieval: dense attention, sparse
attention, grouped-query attention, sliding-window attention with global memory,
or other recent attention modifications can in principle replace their token
memory with a concept graph memory.

The theoretical part of the paper explains why concept creation is a structural
operation rather than a cache heuristic.
A runtime state induces an information structure: two cases that map to the
same runtime cell must share the same local payload unless the representation is
refined.
Repair changes the payload attached to an existing cell; refinement changes the
cells themselves by adding a distinction.
For a loss $L$, the excess risk decomposes as
\[
R_L(f_t)-R_L(\Pi^\star)
=
\underbrace{R_L(f_t)-R_L(\Pi_t)}_{\Delta_{\rm repair}}
+
\underbrace{R_L(\Pi_t)-R_L(\Pi^\star)}_{\Delta_{\rm refine}}.
\]
The first term can be reduced by better values inside the current concept
graph.
The second requires changing the graph: a missing concept or edge means the
runtime state is too coarse for the consequence being predicted.
Under log loss, the value of adding a distinction is $I(Y;U\mid\Pi)$; under
squared loss, it is the corresponding conditional-variance reduction.

The paper makes three contributions.
\begin{itemize}
    \item \textbf{Architecture.} We introduce Concept Graph Attention, a causal
    mixer that reads continuously from a dynamic concept graph and updates that
    graph with gated writes, trainable retention decay, and local forward-time
    memory operations.
    \item \textbf{Theory.} We formalize repair versus refinement for runtime
    information structures and relate concept creation to consequence-valued
    information gains.
    \item \textbf{Evaluation.} We implement CGA in the PoST/Zoology framework
    and evaluate it as an attention-like sequence mixer on state-equalized
    multi-query associative recall, with language-model experiments as the next
    scaling target.
\end{itemize}

The claim boundary is important.
CGA is not presented as a fully optimized replacement for every attention
kernel.
It is a method proposal for changing the memory basis of attention from tokens
to runtime concepts.
The current implementation trains slow projections by backpropagation, while
the concept graph update itself is a forward-time memory operation.
This places CGA between attention, external memory, and efficient recurrent
mixers: it uses differentiable content-addressed reads like attention, but the
state being read is a compact, updateable concept graph rather than the entire
prefix token table.
